## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, chip inner horizontal padding, definition-list row gap |
| sm | 8px | Inline padding in inputs, between table cells, label-to-value gap |
| md | 16px | Default element spacing, card body padding, nav item padding |
| lg | 24px | Section padding within a view, health strip vertical padding |
| xl | 32px | Between card blocks on drilldown pages |
| 2xl | 48px | Major section breaks (e.g. between query console and table) |
| 3xl | 64px | Page-level vertical rhythm (top/bottom page padding) |

Exceptions:
- Health strip height: 40px (fits aggregate pill + service dots in one row without crowding)
- Touch / click targets: minimum 32px tall for all interactive elements (table rows, chip buttons, nav links)
- Code hash chip in list view: 8-char truncated, displayed inline with 4px padding — no min-height rule; not a primary interactive element

## Typography

| Role | Size | Weight | Line Height | Font Stack | Usage |
|------|------|--------|-------------|------------|-------|
| Body | 14px | 400 (regular) | 1.5 | sans-serif stack | Table cell content, definition list values, paragraph text |
| Label | 12px | 600 (semibold) | 1.4 | sans-serif stack | Table column headers, definition list keys, chip labels, nav items |
| Heading | 20px | 600 (semibold) | 1.2 | sans-serif stack | View page title (e.g. "Tools", "Execution History") |
| Display | 28px | 600 (semibold) | 1.15 | sans-serif stack | Reserved for the main page `<h1>` brand/app name in shell header only |

Mono rendering: code_hash values, raw source `<pre>`, query console input, and timestamps all use **Body size (14px) with the monospace font family (`--font-mono`)**. No separate mono size token. Line-height 1.6 applied via a `.font-mono` utility class on `<pre>` blocks for comfortable vertical rhythm. The monospace font-family provides the visual distinction; a separate 13px size is not needed.

Rules:
- Body (14px) is the floor — nothing in the UI renders below 12px (Label).
- Two weights only: 400 and 600. No 500, no 700.
- `<pre>` blocks use `font-family: var(--font-mono); font-size: var(--text-body); line-height: 1.6;`.

## Color

All three primary tokens are locked by REQUIREMENTS.md CONSOLE-02 and verified against
`forge_dark_theme.py`.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#242424` | Page background, main content area, table row background |
| Secondary (30%) | `#2a2a2a` | Card surfaces, nav sidebar/topbar background, drilldown panel backgrounds, health strip background, `<pre>` block background |
| Accent (10%) | `#cc9c00` | Accent reserved-for list below |
| Accent muted | `#664e00` | Selection highlight in inputs; active nav item background |
| Text primary | `#cccccc` | All body and label text |
| Text bright | `#ffffff` | Heading text, active chip label, column header text |
| Text muted | `#999999` | Placeholder text, secondary metadata, "Last refreshed Xs ago" label |
| Border subtle | `#333333` | Card borders, table row dividers, input borders (unfocused) |
| Border medium | `#555555` | Button borders, focused input outline default |
| Destructive | `#4d0000` | Not used in Phase 10 (read-only phase). Reserved token defined for future phases. |
| Status: ok | `#2F5F00` | Health dot: service reachable |
| Status: warn | `#664e00` | Health dot: service degraded (amber family) |
| Status: error | `#660000` | Health dot: service unreachable |

**Accent (`#cc9c00`) is reserved for:**
1. Focus ring on form inputs (query console input, any `<input>` or `<select>`) — matches forge_dark_theme.py `QLineEdit:focus`
2. The value column text in the tool drilldown 2-column provenance definition list (D-14: "amber accent on value column")
3. Active/current nav item left border indicator (2px solid amber border-left)
4. Query console error token highlight — unparseable tokens are underlined or background-tinted in `#664e00` with amber text (D-10)
5. "Refresh" button border-bottom indicator (subtle, not fill) to distinguish it from neutral buttons
6. Aggregate health status pill border when status is "degraded"

Accent is NOT used for: table row hover, general link styling, pagination buttons, chip hover states.

## Component Inventory

Phase 10 introduces these hand-written HTML/CSS components (no third-party component library). All are implemented as Jinja2 macros or partials.

**Primary visual anchor on list views:** the query console input + preset chip row. **Primary visual anchor on drilldown pages:** the provenance definition list card (two-column, amber value column).

### Shell (shell.html)

- **Top nav bar**: `#2a2a2a` background, 40px height, app name in Display (28px/600), five nav links in Label (12px/600), active link has 2px amber left border (vertical nav item style) or bottom border (horizontal). No logo/icon.
- **Health strip**: 40px persistent strip anchored to bottom of top nav or top of `<main>`. Contains: one aggregate status pill (rounded-4px, border styled by status color) + seven service micro-dots (8px × 8px circles, one per service from D-18). Each dot has a `title` tooltip with the service `detail` string. Click pill → Alpine `x-show` expand to per-service detail panel inline.
- **Main content slot** (`<main id="view-main">`): Receives htmx swaps. Full-page bg `#242424`.

### List views (tools, execs, manifest)

- **Query console**: Full-width `<input>` in monospace font (14px, `--font-mono`), `#3a3f4f` background, border `#555555`, focus border `#cc9c00`. Below input: 3-4 preset chip buttons. Error message appears as 12px amber text directly below input — never as a toast (D-10).
- **Preset chips**: Rounded-2px buttons, bg `#2a2a2a`, border `#555555`, Label font (12px/600). Hover bg `#333333`. No amber fill — amber is not used for chip hover. One-click stuffs query and submits.
- **Refresh button + staleness label**: Right-aligned above table. "Refresh" button neutral style (bg `#4a4a4a`, border `#555555`). Staleness label in Text muted (12px/400).
- **Data table**: Full-width, no outer border. Column headers: Label (12px/600), `#ffffff`. Table row: body (14px/400), `#cccccc`. Row hover bg `#2a2a2a`. Row border-bottom `1px solid #333333`. code_hash column: monospace font (14px, `--font-mono`), first 8 chars + hover tooltip for full hash (D-16).
- **Pagination**: Prev / Next buttons + "Page N of M" label centered below table. Neutral button style. No numbered page list in v1.3.

### Drilldown pages (tool detail, exec detail)

- **Provenance card**: `#2a2a2a` background, `1px solid #333333` border, `border-radius: 4px`, padding `md` (16px). Two-column definition list: key column Label (12px/600) in `#999999`; value column Body (14px/400) in `#cc9c00` (amber). Monospace font (14px, `--font-mono`) for `code_hash` and `synthesized_at` values.
- **Consumer-tag chip row**: Inline row of read-only chips below provenance card. Same chip style as preset chips but non-interactive (no hover, no click). Label (12px).
- **Raw source `<pre>`**: Monospace font (14px, `--font-mono`), line-height 1.6, `#1a1a1a` background (darker than secondary for code contrast — adopted from forge_dark_theme.py `QTextEdit`), `1px solid #333333` border, `border-radius: 2px`, padding `sm` (8px). No JS syntax highlighter (D-17). "Copy source" button: top-right corner of pre block, neutral button style, Alpine `@click` handler (~4 LOC per D-14).
- **Raw sidecar JSON**: `<details>` element, default `closed`, same pre styling. Label reads "Show raw sidecar JSON (engineer mode)". "Copy JSON" button: top-right corner, same neutral style as copy source button.

### Health dedicated view (/ui/health)

- Per-service card grid: 2- or 3-column grid. Each card: `#2a2a2a` bg, status color dot, service name in Label, detail text in Body. Auto-refreshes at 5s cadence (tab-visibility paused per D-20/22).

### Chat stub (/ui/chat)

- Centered card: `#2a2a2a` bg, padding `xl` (32px), border-radius 4px. Heading (20px/600): "LLM Chat". Body copy per copywriting contract below. Two chip-links pointing to relevant query presets.

## Interaction Contracts

### htmx swap targets

| Trigger | hx-target | hx-swap | hx-push-url |
|---------|-----------|---------|-------------|
| Nav link click (hx-boost) | `#view-main` | `innerHTML` | `true` |
| Chip click (query submit) | `#view-main` | `innerHTML` | `true` |
| Query console Enter | `#view-main` | `innerHTML` | `true` |
| Pagination prev/next | `#view-main` | `innerHTML` | `true` |
| Table row drilldown click | `#view-main` | `innerHTML` | `true` |
| Health strip poll (10s) | `#health-strip` | `outerHTML` | `false` |
| Health dedicated view poll (5s) | `#health-view-content` | `innerHTML` | `false` |
| Pill click (expand) | Alpine local state only | n/a | `false` |

### Keyboard affordances

| Key | Action | Scope |
|-----|--------|-------|
| `/` | Focus query console input | Any view with a query console; global shortcut |
| `Enter` | Submit query | Query console input focused |
| `Esc` | Clear query console input and reset to unfiltered view | Query console input focused |

No other keyboard shortcuts in v1.3 (D-11; modifier chords and `g t`-style nav deferred).

### State encoding

All filter and pagination state is encoded in plain URL query params. No fragment hash state (D-27). On full-page load, the query console input is pre-populated server-side from the URL params (D-26). `hx-push-url="true"` on every state-changing interaction.

### JS-disabled graceful degradation

All read views render complete HTML on first paint. Filter links and pagination links degrade to standard `<a href>` GET requests. The query console input submits via `<form method="GET">` fallback. JS-disabled is not a certified UAT path but must not produce broken pages.

## CSS Architecture

Phase 10 ships one handwritten CSS file: `forge_bridge/console/static/forge-console.css` (target ~4 KB per D-32).

CSS custom properties declared on `:root` (web idiomatic translation of the three locked tokens):

```css
:root {
  --color-bg:          #242424;  /* Dominant 60% */
  --color-surface:     #2a2a2a;  /* Secondary 30% */
  --color-surface-deep:#1a1a1a;  /* Code block bg */
  --color-accent:      #cc9c00;  /* Amber 10% */
  --color-accent-muted:#664e00;  /* Selection, active */
  --color-text:        #cccccc;  /* Body text */
  --color-text-bright: #ffffff;  /* Headings */
  --color-text-muted:  #999999;  /* Secondary text */
  --color-border:      #333333;  /* Subtle borders */
  --color-border-md:   #555555;  /* Medium borders */
  --color-status-ok:   #2F5F00;
  --color-status-warn: #664e00;
  --color-status-error:#660000;

  --font-sans: "Segoe UI", "Helvetica Neue", Arial, sans-serif;
  --font-mono: "Consolas", "Monaco", "Courier New", monospace;

  --text-display: 28px;
  --text-heading: 20px;
  --text-body:    14px;
  --text-label:   12px;

  --space-xs: 4px;
  --space-sm: 8px;
  --space-md: 16px;
  --space-lg: 24px;
  --space-xl: 32px;
  --space-2xl:48px;
  --space-3xl:64px;
}
```

`--font-mono` is a font-family token only. Mono content uses `font-size: var(--text-body)` (14px). No `--text-mono` size token exists.

No Tailwind. No CSS framework. No `@import` from external URLs. All styles in a single flat file.
