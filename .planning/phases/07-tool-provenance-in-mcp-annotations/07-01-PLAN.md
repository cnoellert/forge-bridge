## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Pre-synthesis hook → synthesizer | Consumer-supplied `ctx.tags` strings reach the sidecar writer. Eventually reach LLM via `tools/list` rendering (Plan 07-03). |
| Synthesizer → filesystem | `.sidecar.json` written to `~/.forge-bridge/synthesized/<stem>.sidecar.json`. Path is derived from synth tool name (already validated by manifest / watcher — reuse existing guard). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | `.sidecar.json` on disk | accept | Sidecar is co-located with manifest-verified `synth_*.py`; any post-hoc edit is caught by the watcher's manifest hash check on the `.py` file (Phase 6-02). Staleness of `.sidecar.json` vs runtime annotations is an accepted non-goal (PITFALL P-02.4, SUMMARY "staleness accepted"). |
| T-07-02 | Information Disclosure | `ctx.tags` written to sidecar (plaintext) | accept (this plan) / mitigate (Plan 07-02) | Plan 07-01 WRITES whatever the hook provides; size-budget + redaction-allowlist enforcement is Plan 07-02's `_sanitize_tag()`. This plan's envelope preserves the raw strings so Plan 07-02 can apply the sanitization boundary at READ time (defense-in-depth — writer is additive, reader is authoritative). |
| T-07-03 | Spoofing | `meta["forge-bridge/origin"]` claim | mitigate | Hardcoded literal `"synthesizer"` in the writer; cannot be overridden by consumer hook. Builtin tools get no sidecar (Plan 07-03 maps absence → no `_meta` provenance). |
| T-07-04 | Repudiation | Who wrote the sidecar | mitigate | `meta["forge-bridge/synthesized_at"]` + `forge-bridge/version` + `forge-bridge/code_hash` provide audit trail. All three are hardcoded-computed, not consumer-supplied. |
| T-07-05 | Denial of Service | Unbounded `meta` payload size | accept (this plan) / mitigate (Plan 07-02) | All five canonical meta values are bounded: `origin` (literal), `code_hash` (64 chars), `synthesized_at` (~30 chars), `version` (~15 chars), `observation_count` (int). The 4 KB / 16-tag ceilings apply to consumer-supplied data and land in Plan 07-02 at READ time. |
</threat_model>
