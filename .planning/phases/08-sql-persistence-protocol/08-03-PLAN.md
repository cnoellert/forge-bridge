<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local filesystem → GitHub remote | git push + gh release create transmit commit content + artifact bytes. Repo is public; no secret material is in the wheel/sdist. |
| GitHub tag → projekt-forge's `@v1.3.0` pin | pip resolves the git URL and fetches the tag. Tag tampering on GitHub would let an attacker substitute content. Mitigation: annotated tag + pushed-to-origin identity check. |
| projekt-forge pip install → forge conda env | `pip install -e .` resolves the direct-reference dependency; if the tag were replaced mid-install, code would flow in unsigned. |
| Alembic upgrade → projekt-forge admin PG | DDL runs with the connection role's privileges. Migration file is in version control; no untrusted input. |
