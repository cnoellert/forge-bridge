# Phase 20 — Track A INSTALL.md Walk-through (Author-walked, with deviation per D-02)

> ## ⚠ HEADLINE FINDING — read this first, every time
>
> **The install procedure as documented in `docs/INSTALL.md` is NOT shippable to artists.**
>
> A normal Flame artist — the actual target user of forge-bridge — could not complete this install. Step 3 alone requires comfort with: Linux package managers, `systemctl`, Postgres cluster initialization (`postgresql-setup --initdb`), the `postgres` OS user / `sudo -u postgres` idiom, `pg_hba.conf` syntax + auth-method semantics (ident vs scram-sha-256 vs md5), Postgres `password_encryption` versioning history, and SQLAlchemy/Alembic error decoding. The architecture is sound (chat endpoint cross-host to assist-01 Ollama works end-to-end). The procedure-as-prose is the broken artifact.
>
> Every reader of this UAT should leave with one sentence: **`docs/INSTALL.md` works only for someone who could write it. That is the opposite of what an install doc is for.**

## Walk-through

### Step 1: Prepare the conda environment
Ran `conda create -n forge python=3.11 -y` and `conda activate forge`. Output: as expected — env created, prompt changed to `(forge)`. No deviation. **No gap.**

### Step 2: Install forge-bridge

### Step 3: Set up Postgres
**Most gap-dense step of the walk.** Six gaps surfaced (see Gap log #1–#6).

- 3a (Confirm Postgres installed/init/running): the doc as authored at walk start did NOT have a 3a sub-step (gap #1, #3 → patched inline by author at HEAD `db56f87`). Walker hit "could not connect to template1" — daemon not running.
- After patch landed: `systemctl status postgresql` → `Loaded ... disabled ... inactive (dead)`. `sudo systemctl start postgresql` → failed because cluster never initialized. `sudo postgresql-setup --initdb` initialized `/var/lib/pgsql/data/`. `sudo systemctl enable --now postgresql` → started and enabled on boot. `nc -z localhost 5432` → up.
- 3b (Create the forge user and database): `sudo -u postgres createuser -P forge` (gap #1 — doc didn't say WHO the superuser was; gap #2 — `sudo -u postgres` "could not change directory" warning is benign but doc didn't pre-empt). Password set to `forge` at the prompt. `sudo -u postgres createdb -O forge forge_bridge` succeeded. Re-running showed `database "forge_bridge" already exists` (idempotent — gap noted, harmless).
- 3c (Alembic migrations): first `alembic upgrade head` failed with `FATAL: Ident authentication failed for user "forge"` (gap #4 — stock Rocky `pg_hba.conf` ships with `ident` auth, rejects password connections). Flipped pg_hba `ident → scram-sha-256`, reloaded. Next attempt: `FATAL: password authentication failed for user "forge"` (gap #6 — Rocky cluster has `password_encryption = md5`; scram-sha-256 verification can't read MD5-stored hashes). Flipped pg_hba `scram-sha-256 → md5`, reloaded. Next attempt: alembic migrated successfully. `alembic current` → `0003 (head)`.

### Step 4: Install the Flame hook (Track A)

Walker chose persistence (Option A from author guidance) and wrote `export` lines into `~/.bashrc`. `source ~/.bashrc` reloaded conda init and reverted to `(base)` env (gap #8). `conda activate forge` restored the env; FORGE_* vars confirmed via `env | grep FORGE_`. **2 gaps surfaced (#7, #8).**

### Step 6: Start the server
Ran `python -m forge_bridge` per the doc's "Daily local launch" form. Server printed startup banner, then `Could not connect to forge-bridge at ws://127.0.0.1:9998` (gap #11), then `Client 'mcp_claude' disconnected` and process exited because no MCP client on stdin (gap #10). Tried again with the `tail -f /dev/null | python -m forge_bridge` keepalive form — same 9998 warning. Eventually triaged via SSH and discovered the doc's "all four surfaces in one shot" claim is false: `python -m forge_bridge.server` (the WebSocket bus on `:9998`) and `python -m forge_bridge` (MCP stdio + Console HTTP on `:9996`) are TWO SEPARATE PROCESSES. The bus must be started first. Once both started in correct order, all 4 hosted surfaces (5 with Flame hook) were reachable. **3 gaps surfaced (#10, #11, #12).** Most consequential gaps of the walk.
