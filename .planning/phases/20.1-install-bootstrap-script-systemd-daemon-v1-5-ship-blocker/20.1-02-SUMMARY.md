# Phase 20.1 Plan 02: Systemd Unit Files Summary

**One-liner:** Two systemd unit files for Linux daemon lifecycle — WS bus (`:9998`) and MCP+Console (`:9996`) — with `Requires=` cascade ordering, `StandardInput=null` fix, and 7 regression guards in CI.

## What Was Built

Created `packaging/systemd/forge-bridge-server.service` and `packaging/systemd/forge-bridge.service` — the two Linux systemd unit files that turn forge-bridge into an OS-supervised daemon pair. These close three Phase 20 UAT gaps:

- **Gap #10** (bare `python -m forge_bridge` exits when stdin closes): Both units set `StandardInput=null`, which sends an explicit `/dev/null` FD that FastMCP's stdio handler tolerates instead of interpreting as "client disconnected."
- **Gap #11** (doc lie about single-process launch): `forge-bridge.service` sets `Requires=forge-bridge-server.service` — a hard dependency enforcing that the WS bus is started first and that bus death cascades to Console restart, not a silent degradation.
- **Gap #12** (no clean stop semantics): `Restart=on-failure` on both units delegates stop/restart lifecycle to systemd; operator uses `systemctl stop / restart` instead of manual process management.

Extended `tests/test_packaging.py` with 7 systemd-specific regression tests (total now 14). The Phase 20 gaps now have automated guards that will catch regressions if someone edits the unit files.

## Unit File Key Directives

### forge-bridge-server.service

| Directive | Value | Rationale |
|-----------|-------|-----------|
| `Type=simple` | `simple` | Python `python -m foo` is a foreground process |
| `EnvironmentFile` | `/etc/forge-bridge/forge-bridge.env` | Reads from P1 env template installed by P4 |
| `ExecStart` | `...python -m forge_bridge.server` | Submodule entry point (NOT the orphan top-level file) |
| `StandardInput` | `null` | Phase 20 gap #10 fix — prevents FastMCP stdio exit |
| `Wants=postgresql.service` | soft dep | Phase 8 STORE-06 — Postgres outage must not cascade-fail bridge |
| `User=__SUDO_USER__` | placeholder | Substituted by P4 install-bootstrap.sh |
| `WorkingDirectory` | `/home/__SUDO_USER__` | JSONL writes in `~/.forge-bridge/` invariant |

### forge-bridge.service (additional directives vs bus unit)

| Directive | Value | Rationale |
|-----------|-------|-----------|
| `Requires` | `forge-bridge-server.service` | Hard dep — bus death cascades to Console; Phase 20 gap #11 fix |
| `After` | `forge-bridge-server.service network.target postgresql.service` | Boot ordering — bus starts first |
| `ExecStart` | `...python -m forge_bridge` | Parent module dispatches to forge_bridge.mcp.server.main |
