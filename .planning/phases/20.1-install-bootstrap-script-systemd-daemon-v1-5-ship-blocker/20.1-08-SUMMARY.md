# Phase 20.1 Plan 08: MCP HTTP transport for daemon mode + service-persistence regression test Summary

**One-liner:** Closed flame-01 Linux walk Gap 4 by adding `--transport streamable-http` flag to `python -m forge_bridge`, switching systemd + launchd daemons away from stdio (which exited on EOF), and adding a subprocess liveness regression test.

## What Was Built

### forge_bridge/__main__.py — two new Typer options

- `--transport` (str, default `"stdio"`) — MCP transport mode: `stdio | sse | streamable-http`. Validated against the allowed set; `typer.BadParameter` on anything else. Default `stdio` preserves existing Claude Desktop / Claude Code launch-process config compatibility.
- `--mcp-port` (Optional[int], default None) — D-27 flag > `FORGE_MCP_PORT` env > 9997 precedence. Forwarded to `mcp_main(transport=transport, port=resolved_mcp_port)`.

### forge_bridge/mcp/server.py — main() extended

`main()` now accepts `transport: str = "stdio"` and `port: int = 9997`. Sets `mcp.settings.port = port` before calling `mcp.run(transport=transport)` (port assignment skipped under `stdio` to avoid mutating settings unnecessarily).

### packaging/systemd/forge-bridge.service — daemon-mode fix

- `ExecStart` updated to `python -m forge_bridge --transport streamable-http --mcp-port 9997`
- `StandardInput=null` **removed** — streamable-http transport runs a uvicorn server that does not read stdin; the line was the root cause of Gap 4
- `Restart=on-failure` → `Restart=always` — clean FastMCP exit (code 0) would not trigger `on-failure`; `always` ensures self-healing on any exit

### packaging/launchd/forge-bridge-daemon — same flags on macOS

`exec` line now passes `--transport streamable-http --mcp-port 9997` alongside `-m forge_bridge`.

### packaging/forge-bridge.env.example — 8th key

## Gap Closed

**Gap 4 (flame-01 Linux walk, Plan 20.1-07):** `forge-bridge.service` ran FastMCP in stdio transport with `StandardInput=null`. FastMCP read `/dev/null` → EOF → exited cleanly (code 0). `Restart=on-failure` did not respawn. `:9996` Console + chat + Read API (co-hosted via FastMCP lifespan) became unreachable. Three of five user-facing surfaces were down.

**Fix:** `--transport streamable-http` switches FastMCP to a long-running uvicorn server that ignores stdin completely. `Restart=always` ensures respawn on any exit. `StandardInput=null` removed as it is no longer needed (and was actively harmful for stdio transport).
