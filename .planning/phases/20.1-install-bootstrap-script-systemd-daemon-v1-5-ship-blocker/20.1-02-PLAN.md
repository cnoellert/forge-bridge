forge_bridge ENTRY POINTS (DO NOT confuse with the orphan top-level file):
  python -m forge_bridge          → forge_bridge/__main__.py → MCP server + Console (port 9996)
  python -m forge_bridge.server   → forge_bridge/server/__main__.py → WS bus (port 9998)

  ❌ ANTI-PATTERN: forge_bridge/server.py (top-level FILE, not the directory) is a pre-Phase-5 orphan
     with broken imports. UNIT FILES MUST NOT REFERENCE IT. Use `forge_bridge.server` (which resolves
     to the directory's __main__.py). See CLAUDE.md "Note:" warning.

PLACEHOLDERS (substituted at install time by P4 install-bootstrap.sh):
  __SUDO_USER__   → operator's login user (validated against ^[a-z][a-z0-9_-]*$)
  Conda Python path: hardcoded as /home/__SUDO_USER__/miniconda3/envs/forge/bin/python in the
                     ExecStart line. (Pitfall 4 / Assumption A1 — operators with non-default conda
                     prefix override via FORGE_PYTHON in the env file; ExecStart=${FORGE_PYTHON}
                     would also work but is deferred to P4 because the env-file substitution only
                     fires at install time. For 20.1 P2 ships the literal-path form.)

WHY Wants=postgresql.service NOT Requires=postgresql.service (RESEARCH.md line 383-385):
  Postgres maintenance restarts must NOT cascade-fail forge-bridge — Phase 8 STORE-06 designed
  JSONL-authoritative + SQL-mirror precisely so the daemon survives DB outages. Wants=+After=
  is soft dep + ordering; Requires= would cascade.

WHY Requires=forge-bridge-server.service IS hard:
  Phase 07.1 graceful-degradation only covers the WARN path (Console boots and warns if WS is
  unreachable). At runtime, if the bus DIES, we want the Console to die too and let systemd
  restart both. Requires= is correct here.

WHY StandardInput=null (Phase 20 gap #10 fix):
  FastMCP stdio transport reads stdin. Under systemd Type=simple, stdin is closed by default,
  which FastMCP can interpret as "client disconnected, shut down" — unit goes inactive in <2s.
  StandardInput=null sends an explicit /dev/null FD that FastMCP's stdio handler tolerates.
