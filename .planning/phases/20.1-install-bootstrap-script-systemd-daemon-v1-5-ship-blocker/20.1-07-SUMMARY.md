2. **Linux walk on flame-01** — the first pass surfaced the structural daemon-mode gap (stdio +
   `StandardInput=null` exits cleanly, `Restart=on-failure` doesn't respawn, `:9996` Console
   dies with FastMCP lifespan). Recorded as **FAIL**, routed to Plan 20.1-08 (commit `c068acc`).
   After Plan 20.1-08 shipped (`--transport streamable-http --mcp-port 9997` flag + lifespan
   composition into the Starlette ASGI app, commit `7fe9779`), re-walked Steps 6-8 and confirmed
   PASS-with-deviations: both daemons `active`, `:9996 + :9997 + :9998` all bound, `forge doctor`
   PARTIAL with `daemon_state ok`, no `Deactivated successfully` cycles. The original FAIL transcript
   stays in the UAT file per D-02.1 honesty (no silent route-around, no backfill).

3. **macOS walk on portofino** — surfaced 5 in-flight bootstrap gaps:
   - **Gap 1** (commit `c73227f`): `bootstrap_pg_macos` only knew Homebrew Postgres; Autodesk
     ships pgsql-17 at `/opt/Autodesk/pgsql-17/bin` which the script didn't detect. Fix: glob-detect
     `/opt/Autodesk/pgsql-*/bin` and prepend to PATH before falling back to `brew install`.
   - **Gap 2** (commit `c9aac41`): `chown root:${SUDO_USER}` fails on macOS — Linux's user-private-
     group convention doesn't hold; macOS users have primary group `staff`. Fix: resolve via `id -gn`.
   - **Gap 3** (commit `cb687f8`): launchd daemons hit `EX_CONFIG=78` with empty log dir because
     `/var/log/forge-bridge/` was `root:wheel` mode 755 but plist `UserName=cnoellert` — launchd
     can't open `StandardOutPath` for write before forking, bails before the wrapper even runs.
     Fix: chown the log dir to `${SUDO_USER}:${SUDO_USER_GROUP}`.
   - **Gap 4** (operator action, deferred): default `:9996` port collision between forge-bridge
     and projekt-forge (both default to it). Captured informally; may need its own SEED.
   - **Gap 5** (operator action, walker): stale `forge_bridge==1.3.0` in conda site-packages
     shadowed the editable install; daemon python loaded the OLD version that lacked `--transport`.
     Fix: `pip uninstall -y forge_bridge && pip install -e ".[dev,llm]"`.
   - Plus a structural takeaway: forge-bridge ships an opinionated Postgres bootstrap that
     doesn't fit the Autodesk-Postgres-on-:7533 reality. Patchwork at the script level is
     calcifying. Captured as `SEED-PHASE-20.1-POSTGRES-OWNERSHIP-V1.6+` (commit `ff067c1`).
   - After all 3 in-flight code fixes + 2 operator actions, walk PASSED-with-deviations:
     all 4 reachable surfaces green (Surface 1 not applicable on macOS — Track B per D-08),
     `forge doctor` PARTIAL with `daemon_state ok`, chat round-trip works against assist-01
     Ollama (slow but expected per `SEED-CHAT-STREAMING-V1.4.x`).
