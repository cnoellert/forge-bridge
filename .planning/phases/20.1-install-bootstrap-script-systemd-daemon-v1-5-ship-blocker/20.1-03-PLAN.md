<objective>
Ship the macOS launchd parity surface to the Linux systemd units (P2). Two plists in `packaging/launchd/` plus two wrapper bash scripts that solve launchd's lack of `EnvironmentFile=` analog and lack of `Requires=` ordering.

Purpose: macOS Flame workstations are first-class deployment targets per CONTEXT.md D-04, D-06. portofino is the macOS UAT gate per D-18. Without these plists + wrappers, P4 install-bootstrap.sh has nothing to copy into `/Library/LaunchDaemons/` and `/usr/local/bin/` on macOS, and the macOS UAT walk cannot reach all 5 surfaces.

Output:
- `packaging/launchd/com.cnoellert.forge-bridge-server.plist` (bus daemon registration)
- `packaging/launchd/com.cnoellert.forge-bridge.plist` (MCP+Console daemon registration)
- `packaging/launchd/forge-bridge-server-daemon` (bash wrapper: source env → exec python -m forge_bridge.server)

Why two-plist + wrapper pattern (RESEARCH.md Pattern 2 + line 499): launchd has NO EnvironmentFile= analog and NO Requires= analog. Hardcoding env in plist forces install-bootstrap.sh re-run + bootout/bootstrap on every env edit. The wrapper-script pattern means operator edits `/etc/forge-bridge/forge-bridge.env` and `launchctl kickstart -k system/com.cnoellert.forge-bridge` reloads — same workflow as `systemctl restart` on Linux. Operator parity wins.

  # /usr/local/bin/forge-bridge-daemon
  # macOS wrapper for MCP+Console with bus-readiness gate.
  # launchd has no Requires= analog — the loop below replicates the systemd cascade ordering
  # forge-bridge.service gets via Requires=forge-bridge-server.service on Linux.
  set -euo pipefail

  if [ -f /etc/forge-bridge/forge-bridge.env ]; then
      set -a
      . /etc/forge-bridge/forge-bridge.env
      set +a
  fi

  # Order gate — wait up to 30s for the bus to be reachable on :9998 before exec'ing.
  PORT="${FORGE_BRIDGE_PORT:-9998}"
  for i in $(seq 1 30); do
      if nc -z localhost "$PORT" 2>/dev/null; then break; fi
      sleep 1
  done

  exec "${FORGE_PYTHON:-/Users/__SUDO_USER__/miniconda3/envs/forge/bin/python}" \
       -m forge_bridge
