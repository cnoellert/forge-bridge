## Deviations from Plan

### async helper for registry snapshot (Rule 1 — adaptation)

The PATTERNS.md pattern for `_build_tool_list(n)` used `asyncio.run(_mcp_server.mcp.list_tools())`. With `pytest-asyncio` in `asyncio_mode="auto"` (project-wide setting in `pyproject.toml`), calling `asyncio.run()` inside an already-running event loop raises `RuntimeError: This event loop is already running`. The implementation uses an `async def _list_real_tools_sorted()` helper instead, which works correctly in the pytest-asyncio context.

### completion_tokens recorded as string-length proxy

`complete_with_tools()` returns a `str` — not a token-count object. The bisection records `len(result)` as a proxy for completion tokens. This is sufficient for the bisection's purpose (distinguishing `end_turn` from `loop_budget`). The `prompt_tokens` field remains `None` (not accessible without re-calling the tokenizer from the returned string). This is acceptable for the first-pass sweep.
