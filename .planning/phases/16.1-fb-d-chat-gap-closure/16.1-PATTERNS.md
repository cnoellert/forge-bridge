**Analog #1 — probe shape:** `forge_bridge/console/read_api.py:215-256` (`_check_flame_bridge` + `_check_ws_server`)

```python
# forge_bridge/console/read_api.py:215-238 — Flame HTTP probe
async def _check_flame_bridge() -> dict:
    try:
        async with httpx.AsyncClient(timeout=1.5) as client:
            r = await asyncio.wait_for(
                client.get(self._flame_bridge_url, timeout=1.5),
                timeout=2.0,
            )
        if r.status_code < 500:
            return {
                "status": "ok",
                "url": self._flame_bridge_url,
                "detail": f"http {r.status_code}",
            }
        return {
            "status": "fail",
            "url": self._flame_bridge_url,
            "detail": f"http {r.status_code}",
        }
    except Exception as exc:  # noqa: BLE001 - intentional: never surface
        return {
            "status": "fail",
            "url": self._flame_bridge_url,
            "detail": type(exc).__name__,
        }

# forge_bridge/console/read_api.py:240-256 — WS-server TCP probe
async def _check_ws_server() -> dict:
    # Best-effort TCP reachability -- full WS handshake is expensive.
    import socket
    try:
        url = self._ws_bridge_url
        host_port = url.split("://", 1)[-1]
        host, _, port_s = host_port.partition(":")
        port = int(port_s) if port_s else 9998
        with socket.create_connection((host, port), timeout=1.5):
            return {"status": "ok", "url": url, "detail": "tcp reachable"}
    except Exception as exc:  # noqa: BLE001
        return {
            "status": "fail",
            "url": self._ws_bridge_url,
            "detail": type(exc).__name__,
        }
```

**What to copy:**
- `httpx.AsyncClient(timeout=1.5)` + `asyncio.wait_for(..., timeout=2.0)` double-cap pattern OR `socket.create_connection(..., timeout=1.5)` for TCP-only probes (RESEARCH §2 recommends async-native `asyncio.open_connection` over `socket.create_connection` to avoid blocking the event loop — `_check_ws_server`'s blocking socket is acceptable in `get_health` because health is one-shot per request, but `_tool_filter` is per-chat-call hot path).
- `try/except Exception` swallow with `type(exc).__name__` reporting — never propagate the connection exception, just return `False`.

**What to differ from the analog:**
- Switch from `socket.create_connection` (blocking) to `asyncio.open_connection` (async) — the chat handler is async-native and we cannot block the event loop. RESEARCH §2 sketch (lines 252-263 of 16.1-RESEARCH.md) shows the exact replacement.
- Return a plain `bool` from the probe, not the `{status, url, detail}` health dict — `_tool_filter` only needs reachable-yes-or-no.

_CAPACITY: float = 10.0

def check_rate_limit(client_ip: str) -> RateLimitDecision:

    with _lock:
        # Lazy TTL sweep — every call inspects all buckets.
        stale = [ip for ip, (_, last) in _buckets.items() if now - last > _TTL_SECONDS]
        for ip in stale:
            _buckets.pop(ip, None)
        # ... refill + decision ...

**What to copy:**
- Module-level dict keyed by backend identity, value is `(reachable: bool, expires_at_monotonic: float)`.
- One `_lock` guarding the dict; `time.monotonic()` for clock; constants in module-level `_PROBE_CACHE_TTL_SEC`, `_PROBE_TIMEOUT_SEC`.
- `_reset_for_tests() -> None` test affordance with `_` prefix — `tests/console/test_chat_handler.py:42` already calls `_rate_limit._reset_for_tests()` and the live integration fixture in `tests/integration/test_chat_endpoint.py:77` does the same. Mirror this surface for `_tool_filter._reset_for_tests()`.

**What to differ from the analog:**
- Use `asyncio.Lock` (NOT `threading.Lock`) — this module's hot path holds the lock across an `await` for the probe. `threading.Lock` would block the event loop for the probe's full ~1.5s timeout. `_rate_limit.py` is sync-only inside the lock so threading is fine there; `_tool_filter` is fundamentally async. RESEARCH §2 sketch line 250 confirms `asyncio.Lock`.
- The cache key is the backend identity tuple `(prefix, host, port)` not the IP. Keys are FIXED (only 2 entries: `forge_*` → `:9998`, `flame_*` → `:9999`), so no eviction is needed — the whole "stale sweep" branch from `_rate_limit.py` is dropped. Simply check `expires_at_monotonic` per entry.
