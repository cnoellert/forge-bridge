## Pitfall 7 Mitigation (patches applied)

The plan noted that patching `startup_bridge + shutdown_bridge` alone might not be sufficient. During test execution, **two additional patches were required**:

| Patch | Reason |
|-------|--------|
| `forge_bridge.learning.watcher.watch_synthesized_tools` | Creates an asyncio task that tries to access the filesystem; fails offline |
| `forge_bridge.mcp.server._start_console_task` | Tries to bind port 9996; returns `(None, None)` from AsyncMock |

All four patches are documented in the test module docstring and each test function's patch chain. The `_lifespan` teardown handles `console_task is None` gracefully via the `if console_task is not None` guard, so returning `(None, None)` from the mock is safe.
