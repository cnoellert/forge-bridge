<objective>
Extend FB-C's `LLMRouter.complete_with_tools()` with an optional `messages: list[dict] | None = None`
kwarg (D-02a Pattern B). When the caller passes a structured `messages` list, the coordinator
forwards it directly to `adapter.init_state()` without auto-wrapping `prompt` into a single user
turn. Backwards-compatible — existing `prompt=` callers unchanged. ~20-30 LOC plus 3-4 unit tests.

Purpose: FB-D's `/api/v1/chat` endpoint accepts a native provider-shape `messages` array per D-02.
Without this overload, FB-D would either lossily stitch prior `assistant`/`tool` turns into a
single string (degrading multi-turn agentic chat quality) or diverge wire-vs-internal. Pattern B
closes that gap cleanly. Lands in Wave 1 as a HARD prerequisite for plan 16-04 (chat handler).

Output: Modified `complete_with_tools()` signature + adapter.init_state() call site + 3-4 new
unit tests covering happy path, mutual-exclusion guard, and tool-role round-trip.
</objective>

def init_state(
    self,
    *,
    prompt: str,
    system: Optional[str],
    tools: list,
    temperature: float,
    messages: Optional[list[dict]] = None,    # NEW (D-02a)
) -> dict:
    """Build initial loop state. If messages is provided, use it verbatim;
    otherwise auto-wrap prompt into [{"role":"user","content":prompt}]."""
    initial_messages = messages if messages is not None else [
        {"role": "user", "content": prompt}
    ]
    return {
        "system": system,
        "messages": list(initial_messages),   # COPY — coordinator mutates state.messages
        "tools": tools,
        "temperature": temperature,
    }
