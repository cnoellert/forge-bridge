Create `forge_bridge/learning/synthesizer.py` with:

1. Imports: `ast, hashlib, inspect, logging, os, tempfile` from stdlib; `importlib.util`; `pathlib.Path`; `typing.Optional`; `unittest.mock.AsyncMock, patch`

2. Constants:
   ```python
   logger = logging.getLogger(__name__)
   SYNTHESIZED_DIR = Path.home() / ".forge-bridge" / "synthesized"
   ```

3. Prompt templates (module-level strings):
   ```python
   SYNTH_SYSTEM = """You are a Flame VFX pipeline tool synthesizer.
   Generate async Python MCP tools from observed Flame code patterns.
   Respond with ONLY the Python function definition. No explanations."""

   SYNTH_PROMPT = """This Flame code pattern was observed {count} times.
   User intent: {intent}

   Code pattern:
   ```python
   {code}
   ```

   Write a single async Python function that:
   - Is named synth_<descriptive_name> (synth_ prefix required)
   - Has typed parameters (str, int, float, bool) extracted from the code literals
   - Has a return type annotation (-> str or -> dict)
   - Has a docstring explaining what it does
   - Calls forge_bridge.bridge.execute() or execute_json() internally
   - Contains no module-level imports (put imports inside the function body)

   Output only the function definition."""
   ```

4. `_extract_function(raw: str) -> str`:
   - If `raw` contains triple-backtick fences, extract content between them
   - Strip leading/trailing whitespace
   - Return the cleaned string

5. `_check_signature(tree: ast.Module) -> Optional[str]`:
   - Walk top-level statements, collect `ast.AsyncFunctionDef` nodes
   - If not exactly 1: return None
   - Check name starts with `synth_`: if not, return None
   - Check `node.returns is not None`: if not, return None
   - Check first body statement is `ast.Expr` containing `ast.Constant` (docstring): if not, return None
   - Return the function name

6. `async def _dry_run(fn_code: str, fn_name: str) -> bool`:
   - Write `fn_code` to a temp file
   - Load via `importlib.util.spec_from_file_location` + `module_from_spec` + `exec_module`
   - Get callable via `getattr(module, fn_name)`
   - Build sample kwargs from `inspect.signature(fn).parameters`: `str -> ""`, `int -> 0`, `float -> 0.0`, `bool -> False`, else `None`
   - Patch `forge_bridge.bridge.execute`, `forge_bridge.bridge.execute_json`, `forge_bridge.bridge.execute_and_read` with `AsyncMock(return_value="")`
   - Call `await fn(**sample_kwargs)`
   - Return True on success, False on any exception
   - Always clean up temp file in finally block

7. `async def synthesize(raw_code: str, intent: Optional[str], count: int) -> Optional[Path]`:
   - Lazy import: `from forge_bridge.llm.router import get_router`
   - Build prompt from SYNTH_PROMPT template
   - Call `await get_router().acomplete(prompt, sensitive=True, system=SYNTH_SYSTEM, temperature=0.1)`
   - Catch `RuntimeError` (LLM unavailable) -> log warning, return None
   - Extract function with `_extract_function(raw)`
   - Stage 1: `ast.parse(fn_code)` — catch SyntaxError -> return None
   - Stage 2: `_check_signature(tree)` — if None -> return None
   - Stage 3: `await _dry_run(fn_code, fn_name)` — if False -> return None
   - Check output path: `SYNTHESIZED_DIR / f"{fn_name}.py"`
     - If file exists and `hashlib.sha256(existing.encode()).hexdigest() == hashlib.sha256(fn_code.encode()).hexdigest()`: log info "identical", return existing path (idempotent)
     - If file exists with different content: log warning "name collision", return None
   - `SYNTHESIZED_DIR.mkdir(parents=True, exist_ok=True)`
   - Write `fn_code` to output path
   - Log info with path
   - Return output path

8. Create `tests/test_synthesizer.py`:
   - All LLM calls mocked with `unittest.mock.AsyncMock` patching `forge_bridge.llm.router.get_router`
   - Use `tmp_path` for synthesized dir, monkeypatch `synthesizer.SYNTHESIZED_DIR`
   - Test all behaviors listed above
   - For _dry_run tests, create actual valid/invalid async function strings and test them
