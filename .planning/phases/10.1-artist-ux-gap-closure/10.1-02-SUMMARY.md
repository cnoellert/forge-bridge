# Phase 10.1 Plan 02: shell.html nav swap fix + UI-SPEC lockstep update — Summary

Ships the one-line template fix (D-37 Option A) that eliminates the 2026-04-23 duplicate-nav render bug and updates `.planning/phases/10-web-ui/10-UI-SPEC.md` in lockstep (D-38) so the UX contract matches the shipped code.

## What Changed

### 1. `forge_bridge/console/templates/shell.html` — line 7 (Task 1, commit `03d4679`)

**Before (Phase 10 shipped state):**

```html
  <nav class="top-nav" hx-boost="true" hx-push-url="true" hx-target="#view-main" hx-swap="innerHTML">
```

**After (Phase 10.1-02 target state):**

```html
  <nav class="top-nav" hx-boost="true" hx-push-url="true">
```

Only attribute deletions: `hx-target="#view-main"` and `hx-swap="innerHTML"` removed from the nav element. `hx-boost="true"` and `hx-push-url="true"` retained. No other line in the file was touched (53 lines in, 53 lines out).

With `hx-target` + `hx-swap` gone, htmx-boost falls back to its default behavior: fetch the target URL, diff `<body>`, swap `body outerHTML` from the response. Because every `/ui/*` handler returns the full shell + view per D-01, the rendered DOM after a nav click contains exactly one `.top-nav` and one `#health-strip` — eliminating the 2026-04-23 duplicate-nav render bug.

### 2. `.planning/phases/10-web-ui/10-UI-SPEC.md` — §Interaction Contracts (Task 2, commit `65b877f`)

**Before (row 152 — Phase 10 shipped state):**

```markdown
| Nav link click (hx-boost) | `#view-main` | `innerHTML` | `true` |
```

**After (row 152 — Phase 10.1-02 target state, D-38):**

```markdown
| Nav link click (hx-boost) | default (body) | default (outerHTML of body) | `true` |
```

Note the deliberate prose-vs-literal-value formatting: `default (body)` and `default (outerHTML of body)` intentionally have no backticks (they describe htmx's default fallback rather than literal attribute values); the `true` cell keeps its backticks (it IS a literal attribute value). This mixed style is explained by the new paragraph inserted directly below the table:

```markdown
**Nav swap note (Phase 10.1, D-37 Option A):** the top-nav on `shell.html` sets `hx-boost="true" hx-push-url="true"` and intentionally omits `hx-target` / `hx-swap`. htmx-boost's default behavior then applies — the full response body is fetched and swapped in place (body `outerHTML`). Because every `/ui/*` handler returns the full shell + view per D-01, the rendered DOM after a nav click contains exactly one `.top-nav` and one `#health-strip`. The 2026-04-23 duplicate-nav render bug came from the earlier `hx-target="#view-main" hx-swap="innerHTML"` combination, which injected the whole shell inside the existing `#view-main`. Removed in Phase 10.1 plan 10.1-02.
```

All seven sibling table rows (Chip click, Query console Enter, Pagination, Table row drilldown, Health strip 10s poll, Health dedicated view 5s poll, Pill click) are byte-identical to the pre-edit state, per D-38's "no other rows in that table are touched" directive. Line-count delta: +2 (one blank line + the paragraph; the row itself is a substitution).
