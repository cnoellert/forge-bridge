<objective>
Pin the Tools-view Status-column data contract **before** any template touches the Status column.

**Critical finding from code inspection (PATTERNS.md "No Analog Gap Flag" + planner read):**
- `ToolRecord` (`forge_bridge/console/manifest_service.py:39-87`) has NO `quarantined` field.
- `ProbationTracker.quarantine()` (`forge_bridge/learning/probation.py:71-88`) *moves* the source file to `~/.forge-bridge/quarantined/` AND calls `mcp.remove_tool(tool_name)`.
- The watcher's scan loop (`forge_bridge/learning/watcher.py:278-293`) mirrors any MCP removal into `manifest_service.remove(stem)` — so a quarantined tool is deleted from `ManifestService._tools`.
- Therefore `ConsoleReadAPI.get_tools()` (`read_api.py:111-113`) never returns a quarantined tool under the current read path.
- The existing preset chip `{"label": "Quarantined", "tokens": "origin:synthesized q:quarantined"}` in `ui_handlers.py:72` is misleading — `_filter_tools()` has no `q:quarantined` handler; the string `"q:quarantined"` would be substring-matched against the tool *name*, not any status field.

**Triage outcome: (b) from CONTEXT `<specifics>`** — no signal exists because quarantined tools are filtered out of the registry. Consequences:
1. **D-40 chip variant set reduces from 3 to 2 in Phase 10.1:** Plan 10.1-03 ships `active` and `loaded` only. The `quarantined` variant CSS may ship for future use but must not be referenced by any template path in this phase.
2. **The bogus `Quarantined` preset chip MUST be removed** from `_tools_preset_chips()` in this same plan — leaving it would invite the re-UAT operator to click a filter that always returns empty, re-failing D-36 for the same reason.
3. **A read-API follow-up** to expose quarantined tools (separate zombie category in `get_tools()` or a new `get_quarantined_tools()` method) is out of scope for Phase 10.1 — logged as Deferred Idea, not done here.

This plan ships exactly two artifacts: (a) a pinning test that locks the contract, (b) the preset-chip cleanup edit. Everything else (Status column, caption, CSS variants, Playwright test) depends on this plan shipping first.
