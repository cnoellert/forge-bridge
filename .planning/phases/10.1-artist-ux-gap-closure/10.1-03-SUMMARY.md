### Decision table (input -> output) for the 4 test cases

| # | origin | code_hash | observation_count | Expected | Shipped |
|---|--------|-----------|-------------------|----------|---------|
| 1 | synthesized | "a"*64 | 5 | `active` | `active` |
| 2 | synthesized | "b"*64 | 0 | `loaded` | `loaded` |
| 3 | builtin | None | 10 | `loaded` | `loaded` |
| 4 | synthesized | None | 3 | `loaded` | `loaded` |

Implementation is the minimal branching that matches:

```python
if tool.origin == "synthesized" and tool.code_hash and (tool.observation_count or 0) > 0:
    return "active"
return "loaded"
```

Signature is ToolRecord-only — no dict branch, no isinstance checks. The sole call site (`ui_tools_handler` TemplateResponse comprehension) iterates `filtered: list[ToolRecord]` before `.to_dict()` is applied, so the helper never needs to disambiguate between input shapes.
