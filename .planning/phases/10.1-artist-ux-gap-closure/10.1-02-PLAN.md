<objective>
Ship the one-line template fix (D-37 Option A) that eliminates the 2026-04-23 duplicate-nav render bug, and update `UI-SPEC.md` §"Interaction Contracts" in lockstep (D-38).

**Recommendation rationale (per CONTEXT `<specifics>` and PATTERNS.md):** Option A (remove `hx-target` + `hx-swap` from the nav; keep `hx-boost` + `hx-push-url`) is chosen over Option B (HX-Request content negotiation) for three concrete reasons:
1. **Scope match:** Option A is one template attribute-tuple edit. Option B requires a new `view_only.html` base + five `*_view.html` template variants + a conditional branch in each of five handlers in `ui_handlers.py`. Phase 10.1 is a gap-closure phase — smallest reversible change wins.
2. **No regression surface:** `hx-boost`'s default swap (whole body from the fetched response, which per D-01 is always the full shell + view) produces the identical rendered output as a full-page navigation. Every existing test of first-paint content (`tests/test_ui_js_disabled_graceful_degradation.py`) still passes because the server response is byte-identical to before.
3. **UI-SPEC simplification:** the swap-table row becomes simpler (no `hx-target` / `hx-swap` to document) rather than more complex (content-negotiation behavior with two branches).

Option B would only win if there were a concrete requirement to preserve partial-swap performance on nav clicks; CONTEXT confirms no such requirement exists.

This plan touches TWO files. One template line; one UI-SPEC table row. Nothing else.
</objective>
