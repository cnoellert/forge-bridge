## Thesis

C2's live E2E proof relocated the frontier: the substrate is trustworthy and execution is solved, so the
weak point is the path from human language into the substrate. The NLT discovery established that **the
translation layer is not missing — it exists, scattered and implicit, and runs today** (`compile_intent`,
tool filter/narrow, `resolve_query_entities`, `extract_explicit_params`, source/executor routing). v1.13's
job is to **formalize, bound, and validate** that machinery as a first-class subsystem — **not** a greenfield
build, and **not** "make chat smarter." The translation/substrate boundary is the chain-step graph;
translation success and substrate success become *separately measurable* verdicts.

Grounded by evidence, not intuition: the D-series partitioned "rename via chat" into **three defects /
three classes / three owners** (grounding · routing+extraction · contextual-resolution), all visible at the
graph boundary, none in the substrate.

## Phases (measure-first-gated; DT/Creative review hardenings folded)

1. **Formalize** — lock the boundary contract + bound the ~6-component inventory. **Mutations:** chain-step
   graph IS the contract (resolution pre-graph). **Reads:** produce a **contract DECISION, not a doc note**
   (DT/Creative Q2) — either *consolidate reads resolution pre-graph* (uniform single contract, more
   invasive) or *adopt an explicit two-part reads contract* (graph-shape + dispatch-resolution-output,
   preserves topology). Forcing the decision is the deliverable; Phase 3 cannot validate what Phase 1 never
   defines. (Discovery pre-delivered the mutation half.)
2. **Taxonomy** — formalize the five translation failure classes: grounding · routing · extraction ·
   entity-resolution · **contextual/stateful resolution**. (Five is complete; capability-gap stays OUT —
   substrate work.) **But name the capability-gap verdict-pair** (Q1): the oracle must recognize *honest
   decline because capability is absent* as **translation-PASS + substrate-GAP** (R9 = the canonical case),
   distinct from a mis-route (translation-FAIL). "Detect-no-capability-and-decline-honestly" is a translation
   **success criterion** — so Phase 4 never optimizes an honest can't into a hallucinated route.
3. **Validation — split 3a / 3b** (DT Q3; you can't measure-first with an unvalidated instrument,
   `[[feedback-audit-before-overfit]]`; v1.10 carried a 3× measurement-debt pattern):
   - **3a — ground the oracle (the milestone KEYSTONE).** Build a **labeled reference corpus** (correct
     graph + resolved params per NL input — CR.1/E2E are *unlabeled* seeds); **build + validate the
     example-fill detector**, measuring its **false-positive rate** (a value matching a docstring example is
     NOT proof it was lifted — a real sequence may legitimately be `30sec_21`); resolve the **reads two-part
     contract** (Q2) so reads are scored at both points. 3a must land before 3b can gate anything.
   - **3b — run the oracle** → independent **translation-pass / substrate-pass** verdicts + the
     example-fill-vs-grounded frequency that ranks Phase 4.
4. **Quality** — evidence-ranked **smallest slices**, each independent, ordered by 3b's measured frequency:
   - strip liftable *value* examples (keep vocab as enum) — D3's 3-site prefix class + sequence-name sites +
     the cross-tool read-docstring bleed (`[[feedback-rhetorical-position-as-architectural-control-surface]]`);
   - PR20-shadows-compile + `extract_explicit_params` parse fix (defect #2);
   - contextual world-state injection into compile (defect #3 — the contextual-resolution class);
   - the per-param provenance signal (see "one artifact" below) surfaced at the existing gate.

## The oracle is ONE artifact (DT ↔ Creative reconciliation)

DT's Q3 reference-corpus, Creative's operator-facing validation surface, and the Q2 provenance flag are the
**same artifact seen from different sides.** A per-param provenance signal — **`grounded-from-intent` /
`grounded-from-context` / `filled-from-example` / `unresolved`** — is *simultaneously*: validation data
(3b's measurement), oracle output, debugging information, and operator-facing feedback at the preview gate.
**Build one surface, not four.** That one signal is what the example-fill detector emits, what the labeled
corpus scores against, what ranks Phase 4, and what shows the operator *"sequence_name: filled from example,
not grounded"* before they ratify — which is exactly the cut-line doctrine (a value filled from an example
is a fact-that-doesn't-exist presented as grounded). This convergence — one artifact serving measurement,
diagnosis, and operator understanding — is the milestone's healthiest structural sign.

## Guardrails (binding)

- **Decompose → rank → measure-first; smallest useful slices; reject the monolith.** The room's record is
  that grounding *shrinks* scope (C2 got smaller; DI.2 T4 gated out). Phase 3 before Phase 4 is the antidote
  to a never-landing grand framing.
- **Ground the instrument before it gates** (`[[feedback-audit-before-overfit]]`). 3a validates the oracle —
  including the example-fill detector's false-positive rate — before 3b's numbers rank anything. "Build
  validation → *validate the validation* → rank quality," not "build validation → rank quality" (Creative).
  The oracle is itself an instrument that can carry measurement debt (v1.10's 3× pattern); 3a is where that
  debt is paid, and it is the milestone keystone — the measure-first spine is only as trustworthy as 3a.
- **Q2 provenance principle:** shallowest-useful representation (per-param provenance at the existing
  preview gate); measure-first any depth beyond it; reject a speculative confidence-scoring /
  multi-candidate framework built before evidence.
- **Formalize, don't rewrite** (`[[feedback-brief-examples-as-behavioral-reference-shapes]]`): preserve the
  battle-tested topology; harden the weak spots.
