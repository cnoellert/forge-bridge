# Phase 07.1 Plan 01: startup_bridge Graceful Degradation Hotfix (TDD) Summary

## One-liner
Applied the surgical fix promised by the v1.2.0 docstring: `startup_bridge()` now gracefully degrades when the forge-bridge WS server on :9998 is unreachable. TDD discipline: failing regression test committed first as the nyquist gate, then the 9-line fix committed separately.

## Goal
Eliminate the latent deployment-blocking bug where `ConnectionRefusedError` from `ws_asyncio.connect()` escapes `_client.start()` and propagates up through the FastMCP lifespan, crashing `python -m forge_bridge` / `python -m projekt_forge --no-db` on any machine without a :9998 listener. The existing WARNING log already promised graceful degradation ("forge_* tools will fail. flame_* tools still work if Flame is running.") — this plan makes the code honor that promise.

## What Changed

### Task 1 (RED) — commit `bb472c9`
`test(07.1): add failing regression for startup_bridge graceful degradation`

Created `tests/test_mcp_server_graceful_degradation.py` with two tests:

1. **`test_startup_bridge_graceful_degradation_on_dead_port`** — uses `socket.bind(("127.0.0.1", 0))` then close to find a guaranteed-free port, points `FORGE_BRIDGE_URL` at it, calls `await mcp_server.startup_bridge()`, and asserts (a) no exception propagates, (b) `mcp_server._client is None` afterward, (c) the existing "Could not connect to forge-bridge" WARNING fired.

2. **`test_shutdown_bridge_after_failed_startup`** — after the failed-connect scenario, asserts `await shutdown_bridge()` is a no-op (returns without raising).

An autouse fixture resets `mcp_server._client` and `mcp_server._server_started` before and after each test so the graceful-degradation scenarios don't leak state into other tests.

**Nyquist gate verified:** running the test against the v1.2.0 code in `forge_bridge/mcp/server.py` (HEAD at the time of this RED commit) fails with `ConnectionRefusedError: [Errno 61] Connect call failed ('127.0.0.1', <port>)` — the exact bug the fix targets.

**Commit is test-only:** `git diff HEAD~1 HEAD -- forge_bridge/mcp/server.py` returns zero lines.

### Task 2 (GREEN) — commit `8cfcb62`
`fix(mcp): startup_bridge graceful degradation on unreachable :9998`

Modified `forge_bridge/mcp/server.py` `startup_bridge()` function body only:

- Moved `await _client.start()` INSIDE the existing try/except so that both it and `wait_until_connected()` are guarded.
- Added a cleanup block inside the except: nested try/except around `await _client.stop()` (idempotent on partially-started clients — `AsyncClient.stop()` safely handles `_ws=None` and `_recv_task=None`), then `_client = None`.
- Existing WARNING message preserved verbatim.

**Diff size:** 30 lines (9 additions, 2 deletions, plus context) — well under the 60-line cap.

**Commit is fix-only:** `git diff HEAD~1 HEAD -- tests/` returns zero lines.
