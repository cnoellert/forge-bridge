## Decisions Made

### gsd-tools manual-edit fallback invoked

**Found during:** Plan 05 Task 2 Step 1.

**Issue:** PLAN.md specifies running `node "$HOME/.claude/get-shit-done/bin/gsd-tools.cjs" roadmap update-plan-progress "07" "07-04" "complete"` to flip the ROADMAP checkbox. Probing the tool's usage (`node gsd-tools.cjs` with no args) revealed the full command list is `state, resolve-model, find-phase, commit, verify-summary, verify, frontmatter, template, generate-slug, current-timestamp, list-todos, verify-path-exists, config-ensure-section, config-new-project, init, workstream, docs-init` — **no `roadmap` subcommand exists at all**. This is not a format-drift failure; the subcommand was never present in this version of gsd-tools.

**Resolution:** Invoked the PLAN.md-sanctioned fallback: a targeted `Edit` on `.planning/ROADMAP.md` replacing the single 07-04 checkbox line. Used `replace_all: false` with the full original line (not just the `[ ]` character) so the substitution was unique and could not collaterally flip any other checkbox. Verified with `grep -nE '^\- \[x\] 07-04-PLAN'` (exactly one match at line 63) and `grep -nE '^\- \[ \] 07-04-PLAN'` (zero matches).

**Rationale:** The PLAN.md explicitly anticipated this failure mode with "If the command fails (e.g. 'plan not found in phase 07', 'roadmap format drift'): Fall back to a manual edit ... Document in the task summary why the gsd-tools call failed so the pattern can be fixed upstream." Task 2 Step 4's commit message cites the gsd-tools gap so the upstream pattern fix is traceable.

### Explicit listing of every Phase 07.1 commit in 07-04-SUMMARY.md

**Issue:** 07-04's close-out commit graph is unusual — eight commits in a sibling phase directory (07.1) retroactively closed the UAT gate that was nominally 07-04's responsibility. Future investigators searching `git log --oneline phases/07-tool-provenance-in-mcp-annotations` will find only `0987525` (release) and `4b64232` (abandoned UAT handoff), which paints an incomplete picture.

**Resolution:** 07-04-SUMMARY.md's "Commits carrying this plan" section lists all eight Phase 07.1 commits with their subjects. This gives any investigator a complete walkthrough of the release→bug-discovery→hotfix→re-release→re-pin→re-UAT sequence without having to jump between `.planning/` directories.

**Rationale:** Plan-level summaries are the project's primary retroactive explanation layer. Making the commit graph explicit in the summary protects the narrative against git-log noise, future commit rewrites, or directory reorganizations.
