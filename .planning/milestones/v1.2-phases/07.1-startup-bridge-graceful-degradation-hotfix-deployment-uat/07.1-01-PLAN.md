<objective>
Fix the latent deployment-blocking bug in `forge_bridge.mcp.server.startup_bridge()` that causes `ConnectionRefusedError` to escape the MCP server's lifespan when the standalone forge-bridge WebSocket server on :9998 is unreachable. The current v1.2.0 code wraps only `wait_until_connected()` in try/except; `_client.start()` raises first (the websockets connect fails) and propagates up through the FastMCP lifespan, crashing the process. The existing docstring + warning log already promise graceful degradation — this fix makes the code honor that contract.

Purpose: SC1 and SC2 of Phase 07.1. The nyquist gate (SC2) requires a regression test that fails against v1.2.0 and passes against v1.2.1 — without it we have no evidence the fix works. This plan uses TDD: RED test committed first (proves the bug), then GREEN commit with the fix.
Output: (1) `tests/test_mcp_server_graceful_degradation.py` with a failing test committed as RED, (2) `forge_bridge/mcp/server.py` updated with the try/except expansion + `_client = None` nulling, committed as GREEN.
</objective>

```python
async def startup_bridge(
    server_url: str | None = None,
    client_name: str | None = None,
) -> None:
    """Connect to forge-bridge server before serving MCP requests.

    Args:
        server_url: WebSocket URL of the forge-bridge server.
                    Defaults to $FORGE_BRIDGE_URL or ws://127.0.0.1:9998.
        client_name: Identifier for this MCP client.
                     Defaults to $FORGE_MCP_CLIENT_NAME or mcp_claude.
    """
    global _client

    server_url = server_url or os.environ.get("FORGE_BRIDGE_URL", "ws://127.0.0.1:9998")
    client_name = client_name or os.environ.get("FORGE_MCP_CLIENT_NAME", "mcp_claude")

    _client = AsyncClient(
        client_name=client_name,
        server_url=server_url,
        endpoint_type="mcp",
        auto_reconnect=True,
    )

    try:
        await _client.start()
        await _client.wait_until_connected(timeout=10.0)
        logger.info(f"Connected to forge-bridge at {server_url}")
    except Exception as e:
        logger.warning(
            f"Could not connect to forge-bridge at {server_url}: {e}\n"
            "forge_* tools will fail. flame_* tools still work if Flame is running."
        )
        # Best-effort cleanup — AsyncClient.stop() is idempotent on a
        # partially-started client. Swallow cleanup errors so the warning
        # above remains the only user-visible signal.
        try:
            await _client.stop()
        except Exception:
            pass
        _client = None
```

Dependencies from `forge_bridge/client/async_client.py`:
- `AsyncClient.start()` at line 180: calls `_connect()` then schedules `_receive_loop()`. When `_connect()` raises (e.g. `ConnectionRefusedError` from `ws_asyncio.connect()`), no recv task is scheduled; `self._recv_task` is None.
- `AsyncClient.stop()` at line 188: sets `_stopped = True`, closes `_ws` if OPEN, cancels `_recv_task` if present, rejects pending — safe on a partially-started client where `_ws` is None and `_recv_task` is None.
- `AsyncClient.wait_until_connected()` at line 214: awaits `_connected` event with timeout; raises `asyncio.TimeoutError` on timeout.

        """Bind a socket to port 0 (OS assigns a free port), close it, and
        return the port number. This gives us a port that is guaranteed
        NOT to have a listener on it when the test starts."""
