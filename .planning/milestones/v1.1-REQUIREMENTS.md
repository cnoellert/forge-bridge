All 16 requirements validated and shipped. Traceability to phases + final outcomes below.

### Import Rewiring (Phase 5)

- [x] **RWR-01**: projekt-forge adds `forge-bridge @ git+...@v1.1.0` to `pyproject.toml` dependencies (later bumped to `@v1.1.1`); PyPI publish remained out-of-scope so a git-URL pin replaced the `>=1.0,<2.0` form originally imagined

- [x] **RWR-03**: projekt-forge's forge-specific tools (catalog, orchestrate, scan, seed) registered via `register_tools()`
- [x] **RWR-04**: `forge_bridge.__file__` resolves to site-packages (not local directory) verified in CI via autouse conftest guard

### Learning Pipeline (Phase 6)

- [x] **LRN-01**: `ExecutionLog` path is configurable at construction (no hardcoded `~/.forge-bridge/executions.jsonl`) — projekt-forge builds per-project path from `FORGE_PROJECT_ROOT/.forge/executions.jsonl`
- [x] **LRN-02**: projekt-forge startup calls `ExecutionLog.set_storage_callback()` with its own storage callback (original one-liner said `set_execution_callback()`; actual API is the per-instance `set_storage_callback()` added in Phase 6-01 with `ExecutionRecord` frozen dataclass contract)
- [x] **LRN-03**: projekt-forge constructs `LLMRouter` with forge config values and injects into `SkillSynthesizer`
- [x] **LRN-04**: `SkillSynthesizer` supports a `pre_synthesis_hook` (async, additive-only) for prompt enrichment from projekt-forge's DB context; enriched prompt verified end-to-end in live UAT against `qwen2.5-coder:32b` on assist-01

## Requirement Outcomes — Noteworthy Adjustments

- **RWR-01 pin form** — shifted from "pip version range" to "git-URL pin" because PyPI publish stayed out-of-scope. `@v1.0.1` → `@v1.1.0` → `@v1.1.1` documents v1.1's three-release cadence.
- **LRN-02 API rename** — original one-liner referenced `set_execution_callback()`; Phase 6-01 designed `set_storage_callback()` as the per-instance surface after CONTEXT.md §decisions settled on instance-scoped (not module-level) registration. Meaning unchanged; naming tightened.
- **SC #3 scope-reduction (ROADMAP)** — "An execution routed through projekt-forge fires the registered storage callback and the event appears in projekt-forge's storage" was scoped to **projekt-forge's log stream** for v1.1 (`_persist_execution` is a logger-only stub). SQL backend (EXT-03) was deferred with `ExecutionRecord` as the stable contract so the callback body swaps in a later release.
