## What Phase N empirically established

(C5 evidence, `v1.6-PHASE-N-RERUN.md`):

### 1. Typed-port compatibility — incremental, local (C2)

`forge_bridge/graph/ports.py` adds `PortTopology`, `PortContract`,
`ChainWireCompatibilityError`, `validate_chain_wire`, `infer_topology`.
The wire check (`_validate_step_chain_wire` in `_step.py`) fires
immediately before each step's dispatch — Shape A, incremental and local,
not a preflight graph-construction phase. The negative probe confirmed
the check discriminates at **item-type** granularity: `list[segment]`
rejected against `list[IterationResult]` — both `kind: list`, rejection
on `item_type`. The 13th discipline-policy enforcement test encodes
incremental-not-preflight.

### 2. Topology expansion — foreach (C3)

`forge_bridge/graph/foreach.py` adds `ForEachNode`, `parse_foreach_step`,
`is_foreach_step`, `IterationResult`. foreach takes `list[X]`, executes
one body step per item, emits `list[IterationResult]`. Phase N initial
scope is one body step; the 15th discipline-policy enforcement test
(`FOREACH_CHAIN_BODY_NOT_SUPPORTED`) is a maturation gate — it is deleted
or inverted when chain/subgraph body lands in Phase N+. Body steps
dispatch through the existing step pipeline with iteration-scoped context;
foreach is a scoped reuse of the existing executor, not a mini-language.

### 3. Topology convergence — collect (C3 + Step A)

`forge_bridge/graph/collect.py` adds `CollectNode`. collect consumes
`list[IterationResult]`, produces a reconciled manifest. It is
substrate-generic — the 14th discipline-policy enforcement test is a
**new variant**: a static source assertion that `collect.py` contains no
named domain-field strings. Reconciliation policy was probe-surfaced and
resolved in Step A (see below).

### 4. Gate semantics over reconciled structure (C3)

The if-gate operates on collect's reconciled output. The C5 positive
probe confirmed: gate matched on the collected `proposed_changes`,
`execution_state=passed`.

### 5. Per-iteration trace render (C4)

render-layer only, anchored on step kind. The 12th discipline-policy
enforcement test (carry-forward from the 25.3 render-gate discovery)

**Load-bearing throughout:** the chain engine in

## Probe-surfaced substrate discovery — scalar variance

The canonical probe surfaced the collect reconciliation semantic gap.
The framing's reconciliation rules underspecified scalar policy —
"scalar fields: reconcile by compatibility rule" carried a hidden
equal-across-iterations assumption that does not survive contact with
real multi-iteration data, where each iteration's `seg_name`,
`source_name`, etc. are naturally unique.

The room-coined insight, preserved verbatim:

> **Scalar variance is the dominant reconciliation case, not the error
> case.**

Step A (`568a475`) resolved it with the Drop policy: constants kept,
list fields merged, varying scalars dropped from the reconciled
top-level, full per-iteration detail preserved in `iterations[]`.
`ChainWireCompatibilityError` was removed from `collect.py` entirely —
it is reserved for topology/type incompatibility, never value variance.
The C5 positive probe confirmed the policy at N=24: the exact
multi-iteration reconciliation that raised `ChainWireCompatibilityError`
before Step A now reconciles cleanly.

This is the **third probe-surfaced substrate discovery in the v1.6 arc**,
after 25.2's `sequence_name` propagation gap and 25.3's identity
boundary. The canonical-probe-as-substrate-gap-surfacing pattern is now
an established v1.6 instrument.

## Process-discipline learnings

### Writer's-room-format carries authorization weight

Mid-phase, commit `6e28315` was authored and committed by code without
room convergence — a seed, a §11 register row, and a canonical probe
annotation. The root cause was not code overreach in isolation: the
writing room had packaged a framing-supersession recommendation as a
ready-to-ship code-handoff (verbatim commit message, exact artifact
contents, file paths). That format *is* an authorization signal. Code
received something shaped like every prior code-handoff and executed it.

**The learning:** framing-supersession, §11-register changes, and
canonical-probe-text changes must be delivered as proposals in
writing-room voice, with a discrete operator + room sign-off step
*before* any code-handoff format is generated. The format itself carries
authorization weight; a framing-domain change shaped as a code-handoff
will be executed as one. The three-step cadence — writing-room proposal
→ explicit room sign-off → code-handoff — was adopted mid-phase and held
for the canonical probe correction.

**Corollary (patches-as-evidence):** patches transmitted through
conversation context can carry copy-time trailing whitespace. The
receiving agent strips it during pre-commit hygiene and surfaces the
cleanup; content stays preserved. Same class as the markdown-fidelity
fence-nesting issue. Both fold into the same discipline: conversation-
transmitted artifacts need a receiving-side fidelity pass.

### Two-amend archaeology on the canonical probe correction

The canonical probe correction (`d1906f4`) was amended twice before
landing — first amend restored most stripped markdown, second amend
restored the final missing primary-probe code fence. All amends were on
a local, unpushed commit with no downstream dependency; amend was the
correct tool (the git-safety caution against amend applies to shared
history, which this was not). Recorded so the single final hash is not
misread as a single clean first pass.

### 6e28315 disposition

Reviewed in place, kept — the seed content was sound; the gap was
writing-room format, not artifact content. Reverting would have added a
rollback commit without changing substrate state. The process correction
is what mattered, not relitigating the artifact.

## Commit primitive — deferred to Phase N+

The canonical probe originally ended in a bare `commit` step. The Step B
rerun empirically established that bare `commit` is not a substrate
primitive — it falls through to tool selection (70-tool match). `commit`
in 25.2/25.3 worked only as a directive on a verb (`rename shots …
commit`), where the tool was known; bare `commit` after `collect` has no
tool context.

`commit` is a chain-finalization primitive distinct from `collect`:
collect converges iteration results; commit applies chain output back to
the domain. It is deferred to Phase N+ with the structural questions
preserved in `.planning/seeds/SEED-COMMIT-PRIMITIVE-PHASE-N+.md`. The
canonical probe was corrected to end at the gate decision (`d1906f4`).

This is the **second first-class-deferral instance** — after 25.3 → Phase
N deferred foreach itself. The pattern: probe pressure surfaces a
substrate primitive, the current phase narrows its pass criterion
honestly, the next phase receives a named seed with the structural
questions intact.

## Cautionary boundary (carried from 25.0–25.3)

Phase N succeeded for the same reasons its predecessors did: small,
explicit, non-magical, structurally honest. Typed ports arrived as a
consequence of topology-expansion pressure, not as imported abstraction.
The minimum-viable-contract discipline held — Commit 2 was bounded to
what foreach needed, not pre-generalized into a graph type system.

The danger after Phase N is the same danger named at each prior close.
The substrate now has validated topology-expanding execution. That is
not permission to build a graph runtime, a scheduler, or async fanout.
The seam phrase remains load-bearing: **new dispatch path, not a new
executor.** Parallel fanout is v1.7. The commit primitive is Phase N+,
and only when its typed input contract, dispatch target, and domain-apply
semantics are explicit.
