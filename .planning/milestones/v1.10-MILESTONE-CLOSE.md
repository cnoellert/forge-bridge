# v1.10 Authority Invariance — Milestone Close

> **What v1.10 was.** A reframe from the v1.9 dogfood's seam-mismatch finding:
> *the system already knows which tools mutate; it just wasn't acting on it.*
> Invariant: **operator authority + execution eligibility must not depend on
> routing path.** Two symptoms, one seam — ①authority changes with route,
> ②eligibility changes with route — split into **DI.1 (trust) → DI.2
> (usefulness)**, both landed in a single day.
>
> **What this artifact is.** The milestone-level reckoning. The per-phase detail
> lives in `DI.1-CLOSE.md` and `DI.2-CLOSE.md`; this synthesizes what the arc
> delivered, what it honestly did *not*, and what carries forward. Voice binding:
> every closure claim is scope-bound — no paragraph would let a cold reader
> conclude "make chat useful" is done.

  empty. (5 code commits `feb4d47..2fb40a4`.)

milestones now without public-surface movement.

## The thesis-vs-outcome reckoning (the honest core)

v1.10 set out to make authority + eligibility **routing-invariant**. It did. What
it did **not** do — and never scoped to — is make chat *answer* the artist's
questions. The framing named this early: **"make chat useful" is several
milestones, not one.** v1.10 is the first two:

- **Trust (DI.1):** a read can never become a mutation by routing accident. Proven.
- **Usefulness, first increment (DI.2):** the reachable resolver-overmatch class is
  resolved **at the arbitration layer** — R8/R9/R10 narrow to a single tool, live.
  But those reads still **do not answer**; they block one seam down at parameter
  resolution (`shot_id` derivable, unresolved). **DI.2 removed the resolver wall and
  revealed the parameter wall behind it.** That is progress correctly stated at its
  native layer, not a "chat now works" claim.

The discipline that made the arc trustworthy was refusing, at every seam, to let one
layer's work secretly become another's: DI.1 didn't let safety become usability; DI.2
didn't let arbitration become recompilation (it left the param fix untouched).

## Measurement-debt reckoning (the milestone's signature)

The arc was measure-first throughout (DI.1 baseline, DI.2 T1 distribution). The
recurring honesty was that **live instruments kept failing to substantiate
load-bearing claims** — 3× named across CR.1→DI.1→DI.2. v1.10 **retired one**: DI.2's
T1 incidentally **demonstrated DI.1's live mutation-block** (R5/R11 hit the gate 3/3),
which DI.1-CLOSE could only suite-prove (Symptom 2 had masked it). The instrument
matured from "blind to 8 of 9 failures" (CR.1) to a reproducible failure-shape
distribution (DI.2 T1, 33 samples) that reproduced the framing's predicted class split.

## The hidden-context audit (a wrong turn avoided)

Before hardening DI.2's close, Creative's hypothesis — *is the corpus accidentally
grading tools that need unavailable context?* — was tested grounded. It returned
**Outcome A**: the system mostly fails on questions it *should* be able to answer; only
R11 is a clean eligibility failure (DI.1-owned). The audit de-risked the "corpus
contaminated, DI.2 chasing the wrong problem" fear and **named a dormant architectural
seam** without building it: **context-eligibility** — DI.1 asks *safe?*, DI.2 asks
*choosable?*, neither asks *eligible given available context?* Captured cheap, graduation
deferred to data.
