## Decisions Made

- **Helper placed between `_TurnResponse` (line 114) and `_ToolAdapter` Protocol (now line 195)** — needs access to `_ToolCall` (line 86) which it constructs; module-level placement keeps it test-importable and matches the project's leading-underscore module-private convention. The plan's `interfaces` block suggested "module top after `_OLLAMA_KEEP_ALIVE`" but `_ToolCall` is defined later, so the helper had to live below the dataclasses. Plan-compliant — the plan's intent was "module-level, not class method," not "specifically before all dataclasses."
- **Helper returns `None` on every failure mode, never raises** — explicit T-16.2.02-01 mitigation. The cheap pre-filter (`stripped.startswith('{')`) catches plain prose without paying `json.loads` cost; the `try/except (json.JSONDecodeError, ValueError)` handles malformed JSON; type-guards on `name` (must be non-empty str) and `arguments` (defaults to `{}` if not a dict) handle structurally-wrong objects.
- **Tolerated string-typed `arguments`** — the structured path at line 410 already handles this quirk; mirroring it in the salvage path means the helper accepts every shape Ollama might emit, not just the most common one.
- **Composite ref `0:{name}`** — mirrors the existing structured path's `f"{idx}:{name}"` convention at line 416 verbatim. The router treats salvaged and structured tool calls identically per T-16.2.02-06 (repudiation accept disposition); no `salvaged:` marker needed.

## Threat Surface Scan

No new attack surface introduced. The salvage helper is defense-in-depth on a pre-existing data flow (LLM → adapter → router). All STRIDE threats from the plan's threat model are mitigated as designed:

- **T-16.2.02-01 (Tampering, raise on malformed):** mitigated — helper wraps `json.loads` in `try/except (json.JSONDecodeError, ValueError)`, returns `None`. Verified inline (`_try_parse_text_tool_call('not json')` returns `None`).
- **T-16.2.02-02 (Tampering, unexpected types in `arguments`):** mitigated — `if not isinstance(args, dict): args = {}` defensive type-guard at line 162 of the helper.
- **T-16.2.02-03 (Information Disclosure, prompt injection coerces tool call):** mitigated — `router.py:414` rejects unknown tool names; `_sanitize_tool_result()` strips injection markers from results (Phase 15 LLMTOOL-06). The salvage path operates upstream of execution, not bypassing it.
- **T-16.2.02-04 (DoS, pathological text):** accepted — adapter input bounded by Ollama context window + LLMTOOL-03 wall-clock cap (120 s default).
- **T-16.2.02-05 (Privilege Elevation, salvaged tool widens tool set):** mitigated — salvage path can only produce a `_ToolCall`; the router enforces against `registered_names`, which is determined upstream by the chat handler's tool-filter result.
- **T-16.2.02-06 (Repudiation, salvage event invisible):** accepted — by design; structured vs. salvaged are indistinguishable at the `_ToolCall` level. The fix tightens the operator-visible failure mode (no more raw-JSON terminal text); finer-grained forensic tracking is a future v1.4.x concern, not in 16.2 scope.
