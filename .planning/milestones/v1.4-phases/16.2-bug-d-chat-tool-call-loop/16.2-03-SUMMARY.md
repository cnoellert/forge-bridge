## Decisions Made

- **Module-top regex placement (line 95) over in-test-function placement.** Mirrors PATTERNS.md "frozen-style module constants" pattern (`_OLLAMA_KEEP_ALIVE`, `_OLLAMA_TOOL_MODELS` in `_adapters.py`; `_CONTROL_CHAR_RE` in `learning/sanitize.py`). Single compile per process; constant is test-importable for any future cross-test reuse.
- **Anchored regex `^\s*\{\s*"name"\s*:` over a substring match.** T-16.2.03-01 accept disposition: legitimate responses that *contain* JSON ("Here is the result: {...}") are NOT false-flagged because they don't *start* with the JSON shape. The accepted residual risk is documented in the regex comment.
- **Two-pronged D-06 #1 + D-06 #2 over either alone.** T-16.2.03-02 mitigation: even if a future LLM emits leading prose ("Sure, here is the call: {...}") that bypasses the regex anchor, the message-count assertion catches the failure mode. The agentic loop must have iterated for the response to be healthy.
- **REJECTED: D-07 "natural prose detection" heuristic.** Too brittle. A `len(content.split()) >= N` or "looks like prose" probe would false-positive on legitimate one-word answers ("yes", "42", "true"). The two surgical assertions in this plan are precise about the exact symptom of Bug D and tolerant of all other valid response shapes. Documented in the regex comment and at multiple decision points throughout this summary.

## Why D-07 Was Rejected

Per CONTEXT D-07 (verbatim):
> Reject any "natural prose detection" heuristic (e.g., counting word boundaries, asserting non-JSON via heuristic content shape). Too brittle. The two assertions in D-06 are surgical: they fail on the exact symptom of Bug D (JSON-as-terminal-text + no iteration) and don't false-positive on edge-case but legitimate responses (e.g., a one-word "yes" answer to a follow-up).

The Bug D failure mode is structural, not stylistic: the *shape* of the terminal content (JSON object starting with `"name"`) and the *shape* of the message envelope (zero tool turns, only one assistant turn). Both are precisely matchable. Heuristic "is this prose?" detection would either:
1. Be loose enough to admit Bug D shapes (defeats the purpose), or
2. Be strict enough to reject legitimate short answers (false-positives).

The D-06 assertions are the targeted alternative: regex matches the exact JSON shape, message-count assertion checks the loop-iteration evidence. Neither has the brittleness D-07 warns about.
