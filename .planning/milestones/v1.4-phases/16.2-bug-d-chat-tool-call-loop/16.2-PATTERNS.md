**Where Bug D enters the picture:** when qwen2.5-coder:32b emits the tool call as JSON-shaped *text* in `message["content"]` instead of as a structured entry in `message["tool_calls"]`, both `text` ends up containing the JSON string AND `raw_tool_calls` is `[]`. The for-loop at line 401 (`for idx, call in enumerate(raw_tool_calls):`) does nothing, returns an empty `tool_calls=[]` list, and `router.py:435` (`if not response.tool_calls:`) terminates the loop with the JSON-text as terminal content.

**Fix shape — fallback text-parser inserts AFTER the existing parse and BEFORE the `_TurnResponse` construction:**

The fix must be a NEW helper called from inside `send_turn` between the existing parse (line 419) and the usage-token block (line 421). Skeleton (planner picks exact regex / parsing strategy based on captured fixture):

```python

# (start of usage-token extraction).

# Bug D fallback: qwen2.5-coder:32b sometimes emits tool calls as JSON-shaped
# text in message.content instead of structured tool_calls. When the structured

class TestOllamaToolAdapterBugDFallback:
    """Phase 16.2 (D-02 step 2 / D-05): when Ollama emits a tool call as
    JSON-shaped text in message.content (instead of structured tool_calls),
    the adapter salvages it. RED commit lands as Plan 01; GREEN commit lands
    as Plan 02 with the _try_parse_text_tool_call helper.

    See Phase 16.2 D-03 for the failure-mode evidence trail.
    """

    @pytest.mark.asyncio
    async def test_text_content_tool_call_salvaged(self):
        """qwen2.5-coder:32b emits {"name": ..., "arguments": ...} as text
        with empty structured tool_calls. The salvage path produces a
        non-empty _ToolCall list so router.py:435 keeps iterating.
        """
        client = MagicMock()
        client.chat = AsyncMock(return_value=_fake_response_dict(
            content=_OLLAMA_BUG_D_RESPONSE_CONTENT,
            tool_calls=None,  # Bug D: structured field empty
        ))
        adapter = OllamaToolAdapter(client, "qwen2.5-coder:32b")
        state = adapter.init_state(prompt="hi", system="s", tools=[], temperature=0.1)
        resp = await adapter.send_turn(state)

        # The salvage produces a non-empty tool_calls list (non-zero is the
        # core fix; the loop now iterates instead of terminating immediately).
        assert len(resp.tool_calls) >= 1, (
            f"Bug D regression — text-shaped tool call not salvaged. "
            f"Got tool_calls={resp.tool_calls!r}, text={resp.text!r}"
        )
        assert resp.tool_calls[0].tool_name == "forge_tools_read"
        # Salvaged content is consumed — must NOT also surface as terminal text.
        assert resp.tool_calls[0].arguments == {"name": ""}
        assert resp.text == "", (
            "salvaged text must not double-emit as terminal content "
            f"(would re-trigger Bug D); got text={resp.text!r}"
        )
```
