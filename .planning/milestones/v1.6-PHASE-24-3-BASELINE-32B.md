## Per-layer baseline findings

### Layer A (runtime economics) — severe under both cold and warm prefix

- Cold: ≥120s for the first response token (0 tokens emitted in 120s)
- Warm iter 1 @ ~15K tokens: 1.4s (tool_error path, fastest case)
- Warm iter 2 @ ~15K tokens: 13.3s (model response after tool-result absorption)
- Warm iter 3 @ ~22K tokens: 80.4s (Phase 23.1's ~71s/turn shape reasserts)

The cold-prefix amplifier (Phase 24.1 archaeology) and the post-tool-result prompt growth (Phase 23.1 archaeology) compound: even warm-prefix probes blow the 120s budget after 3 iters because each successive turn carries more tool-result history.

### Layer B (termination/convergence) — broken in clean isolation

- Iter 1 picks WRONG tool (`flame_find_media`), returns `tool_error`
- Iter 2 self-corrects to `flame_execute_python`, gets `status=ok` result
- Iter 3 RE-INVOKES the same tool with the same args, gets the same result
- Iter 4 would presumably continue the loop, but budget exhausts first

The model never recognizes "I already got the answer; synthesize the conversational response." This is structurally separate from runtime speed — even with infinite time budget, the loop shape suggests the model would continue tool-calling rather than converge to an operator-readable answer. **Layer B is the residual failure surface 24.3 explicitly does NOT scope structurally (framing §3.2 + §10).** Empirical evidence here makes it a candidate for a separate post-24.3 phase under its own framing.

### Layer C (perceived responsiveness) — 120s of operator silence per probe

Both probes return only at the 120s timeout. No intermediate signal of progress reaches the operator. Streaming UX (framing commit 4, history-grows SSE) is the orthogonal UX-decoupling work that makes the path tolerable; per §4.0, it does NOT change the success criterion.

## What this baseline proves

1. **The substrate correction landed and held.** Doctor green pre and post; warm probe emits 2 status=ok graph events with `node_kind="python"` and `elapsed_ms=16`. The §4.3 case from 24.2 (single emission) generalizes to multi-iter dispatch correctness.

2. **The 32b default model is unambiguously inadequate operationally.** Under cold prefix it can't return even one token in 120s. Under warm prefix it can dispatch correctly but cannot terminate within budget.

3. **The cold-prefix amplifier dominates cold probes.** Phase 24.1's ~115s measurement reproduces here as 120s with 0 tokens emitted — the same shape, sharpened to the exhaustion case.

4. **Layer B is structurally observable on warm probes.** With substrate clean and runtime giving the model 3 attempts, the model still re-invokes the same tool with the same args. This is the cleanest Layer B isolation evidence the project has captured.

5. **The model swap to `qwen2.5-coder:14b` is justified by evidence per framing §5 commit 3 condition.** 32b baseline is operationally insufficient. The conditional decision in framing §5 ("CONDITIONAL on baseline evidence") resolves to: swap proceeds. The 14b probe's first job is to test whether smaller inference speeds bring the warm path under 90s for time-to-readable, AND whether 14b retains affordance-selection capability (it would need to NOT pick `flame_find_media` first like 32b did at iter 1).

6. **Streaming (framing commit 4) is justified independent of the swap decision.** 120s of operator silence is intolerable regardless of which model occupies the runtime layer.
