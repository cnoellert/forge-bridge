## What 25.3 empirically established

### Positional anchoring (Item 18 closed)

`forge_bridge/llm/resolver.py` — `\bcommit\b` and
`\bdry[_ -]?run\b` match-anywhere regexes replaced with
position-aware extraction. Directives (commit, dry_run, format
modifiers) are now valid only in their structural positions within
step text.

The truly pathological case — `rename shots with prefix commit` (lone
trailing token) — no longer fires the commit directive. The fix closes
bounded defect 1 from the 25.2 close cursor.

Item 18 maturation condition: "filter + if/then both landed." Both
have. The heuristic compensation is now removed — not because it was
convenient to remove, but because the graph primitives it was waiting on
exist.

### Generic select (Item 12 closed)

`forge_bridge/graph/select.py` adds `SelectNode`, `parse_select_step`,
and `is_select_step` (anchored `^\s*select\s+`, parallel to
`is_if_step`). Chain wiring via
`_step.py:_maybe_execute_select_step` follows the exact dispatch shape of
filter/if-gate: returns `{result, extracted_context,
tool='graph_select', params}`.

Select is topology-preserving:

```text
filter  list[X] -> list[X]   narrow by predicate
select  list[X] -> list[X]   narrow by identity  <- NEW in 25.3
foreach list[X] -> ...       topology expansion  — Phase N
```

Output is always `list[X]`. Single-item selected collection returns a
single-item collection, not a scalar. This preserves chain uniformity and
defers scalar/collection execution semantics to Phase N.

SelectNode does not guess. Zero matches -> `IDENTITY_NOT_FOUND`.
Multiple matches -> `IDENTITY_AMBIGUOUS`. No fuzzy matching, no semantic
retrieval, no ranking.

Ordered field-try: SelectNode tries canonical identity fields in
declared order and matches on the first present:
`seg_name -> shot_name -> sequence_name -> node name -> clip name`.
This is deterministic without requiring upstream normalization. Upstream
normalization and emitter-declared identity fields are explicitly
deferred (approach B and C from the 25.3 framing) — not rejected.

Select input topology after mutation-preview steps: select operates over
the active execution manifest, not the original upstream collection.
After dry_run + if-gate, identity resolution occurs against
manifest-produced entities (e.g., `proposed_changes` entries).

## Substrate discovery 1 — identity boundary (probe-surfaced)

The canonical probe initially used `genesis_0020` (operator shorthand).
SelectNode returned `IDENTITY_NOT_FOUND`: the manifest identity is
`genesis_0020_source_L01` — the full `seg_name` as produced by the
dry_run step. The shorthand is under-specified.

This is not a SelectNode routing failure. It is the expected consequence
of the 25.3 contract. The invariant held exactly as designed.

The fix (`ba36006`) carries the canonical identity at the
`flame_rename_shots` emission site — scoped to that emission site only,
not generalized to all dry_run-producing tools per anti-scope discipline.
Commit message names this verbatim:

> "This is not generalized proposed_changes normalization. It carries
> identity already available at the flame_rename_shots emission site."

Three relaxation paths deferred explicitly (not rejected):
field-qualified select syntax, emitter-declared identity fields, upstream
manifest identity normalization.

This is the second instance of the canonical-probe-as-substrate-gap-
surfacing pattern within the v1.6 arc. First instance: 25.2 probe
surfaced sequence_name propagation gap. Both were design outputs, not
defects.

## Substrate discovery 2 — render-gate anchor surface

`ba36006` also fixed a structural defect in `_maybe_render_gate_step`:
the function previously anchored on result-metadata presence (`if_gate`
dict in result). The chain engine's context propagation carries
`__previous_result__` forward — so the SelectNode's result inherited
`if_gate` metadata from the prior if-step. The gate render path fired
incorrectly for the select step.

Fix: anchor on step text (`step.lstrip().lower().startswith("if")`)
rather than result-metadata presence. Step kind is the correct anchor
surface; result metadata is not stable across context propagation.

Forward risk for Phase N: the same class of bug will surface during
per-iteration foreach trace rendering. Each iteration's result will
carry upstream metadata via context propagation. The Phase N render layer
must anchor on step kind, not metadata presence. Named explicitly here so
Phase N framing can inherit the pattern.

This fix produced the 11th instance of the discipline-policy enforcement
test pattern:
`test_trace_does_not_render_select_manifest_as_gate_decision` —
negative-space assertion that gate strings do NOT render for a select
step inheriting if_gate metadata.

explicit, non-magical, structurally honest. Two substrate discoveries
surfaced by the canonical probe and both fixed correctly — one at the
emission site, one at the render layer — without widening scope.

not permission to build foreach as a 25.x item. The seed is explicit:
foreach without typed ports produces magic, rigid, or fragile substrate.
