#### Live-server harness excerpt — `tests/test_console_http_transport.py:24-75`

```python
import asyncio
import socket

import httpx
import pytest

from forge_bridge.console.app import build_console_app
from forge_bridge.console.manifest_service import ManifestService, ToolRecord
from forge_bridge.console.read_api import (
    ConsoleReadAPI,
    register_canonical_singletons,
)
from forge_bridge.mcp.server import _start_console_task


def _find_free_port() -> int:
    with socket.socket(socket.AF_INET, socket.SOCK_STREAM) as s:
        s.bind(("127.0.0.1", 0))
        return s.getsockname()[1]


@pytest.fixture
async def console_server(tmp_path, monkeypatch):
    """Spin up a real uvicorn-served ConsoleReadAPI on an ephemeral port."""
    from forge_bridge.learning.execution_log import ExecutionLog

    monkeypatch.setattr(
        "forge_bridge.mcp.server._server_started", True, raising=False,
    )
    monkeypatch.setattr(
        "forge_bridge.mcp.server._canonical_watcher_task", None, raising=False,
    )

    log = ExecutionLog(log_path=tmp_path / "execs.jsonl")
    ms = ManifestService()
    await ms.register(_record("a_tool"))
    register_canonical_singletons(log, ms)
    api = ConsoleReadAPI(execution_log=log, manifest_service=ms)

    app = build_console_app(api)
    port = _find_free_port()
    task, server = await _start_console_task(app, "127.0.0.1", port)
    assert task is not None and server is not None

    try:
        yield port, log, ms, api
    finally:
        if server is not None:
            server.should_exit = True
        if task is not None:
            try:
                await asyncio.wait_for(task, timeout=5.0)
            except (asyncio.TimeoutError, asyncio.CancelledError, Exception):
                task.cancel()
                try:
                    await task
                except Exception:
                    pass
```

Key pattern ingredients to copy:
1. `_find_free_port()` to avoid collisions with a real `:9996` server (line 24-27).
2. `monkeypatch` the MCP server startup gate + watcher task to `None` so `get_health` returns a reasonable shape without a real MCP lifespan (lines 42-48).
3. `_start_console_task(app, "127.0.0.1", port)` to run uvicorn on a background asyncio task. Returns `(task, server)`.
4. Teardown uses `server.should_exit = True` → `await asyncio.wait_for(task, timeout=5.0)` → cancel fallback.

## Shared Patterns

### Handler error handling (applies to all `ui_handlers.py` edits — Option B path only)

**Source:** `forge_bridge/console/ui_handlers.py:138-150`
```python
try:
    tools = await request.app.state.console_read_api.get_tools()
except Exception as exc:
    logger.warning(
        "ui_tools_handler failed: %s", type(exc).__name__, exc_info=True,
    )
    return _render_error(
        request, "errors/read_failed.html",
        "Could not load tools — the console API may be restarting. Refresh to try again.",

    )
```
Every handler in `ui_handlers.py` follows this shape: `try` → `await read_api.get_*()` → `except Exception` → `logger.warning("<handler> failed: %s", type(exc).__name__, exc_info=True)` → `_render_error(...)`. Any Option B content-negotiation branch lives *after* the try/except success path, not inside it.

### Fragment template convention (applies to any new or modified fragment)

**Source:** `forge_bridge/console/templates/fragments/tools_table.html` (whole file).
Fragment templates:
- Do NOT `{% extends %}` — they are bare HTML partials.
- Have a single outermost wrapper element with a stable `id` (here `#tools-table`) that htmx `outerHTML` swaps.
- Use the shared classes from `forge-console.css` (`.data-table`, `.mono`, `.empty-state`, `.row-link`, `.chip`, `.chip-row`) — Phase 10.1 introduces `.status-chip[--variant]` following the same BEM flavor.

### CSS custom-property-first styling (applies to all CSS edits)

**Source:** `forge_bridge/console/static/forge-console.css:1` (`:root` block).
Every color and spacing value references a `--color-*` or spacing variable, not a hard-coded hex. Phase 10.1 status-chip variants MUST use `var(--color-status-ok)`, `var(--color-accent)`, `var(--color-accent-muted)`, `var(--color-text)`, `var(--color-border-md)` — no new hex literals.

### Handler-template context shape (applies to any view that re-reads `tools_table.html`)

**Source:** `forge_bridge/console/ui_handlers.py:153-168` (`ui_tools_handler` context dict). The template receives:
```python

    "active_view": "tools",
```

## No Analog Gap Flag

**D-40 `quarantined` chip state has no direct `ToolRecord` field.**
- `ToolRecord` (`manifest_service.py:39-87`) fields: `name, origin, namespace, synthesized_at, code_hash, version, observation_count, tags, meta`. No `quarantined: bool`, no `status: str`.
- The probation module (`forge_bridge/learning/probation.py`) *moves* source files to `~/.forge-bridge/quarantined/` and calls the MCP registry to *remove* the tool — after which the `ManifestService` also drops it (watcher contract). So a quarantined tool may not appear in `get_tools()` at all.
- The preset-chip entry `{"label": "Quarantined", "tokens": "origin:synthesized q:quarantined"}` (ui_handlers.py:72) is a **string-name-substring** filter, not a status-field filter — it matches if the word "quarantined" happens to be in the tool name. This is not a real quarantine signal.

**Planner action required:** before drafting Plan 10.1-02 (Tools-view Status chip + header relabel), verify whether a quarantined tool is ever actually visible in the Tools view under the current read path. If not, the `quarantined` chip variant may never be exercised in practice — at which point D-40's three-value chip (`active` / `quarantined` / `loaded`) reduces to a two-value chip (`active` / `loaded`) and the planner flags the gap for a read-API-level fix (not a template fix) as a separate follow-up before Phase 10.1 closes.
