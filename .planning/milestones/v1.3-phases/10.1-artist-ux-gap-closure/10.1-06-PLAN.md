<objective>
Fix the chip-click shell-duplication bug surfaced in the 2026-04-24 Phase 10.1 pre-UAT smoke check and add a Playwright regression test so it cannot recur.

**Root cause:** `forge_bridge/console/templates/fragments/query_console.html` `submit()` does:

```js
htmx.ajax('GET', url, { target: '#view-main', swap: 'innerHTML', pushUrl: true });
```

where `url` is a full-page route (`/ui/tools?origin=synthesized`). The server returns the full shell+view; htmx stuffs it inside `#view-main`, producing shell-inside-shell. Same D-39 invariant violation as the 2026-04-23 nav-click bug that Plan 10.1-02 fixed â€” but via a different code path (JS `htmx.ajax`, not `hx-boost`) that Plan 10.1-04's Playwright regression did not cover.

**Secondary finding:** `Active synth` preset chip tokens are `origin:synthesized` â€” 4 tools match (3 active + 1 loaded synth). Label promises "active", tokens deliver "synthesized". Rename label to `Synth only` for accuracy; token grammar unchanged.

**Why this is gap closure, not a scope expansion:** the bug blocks Plan 10.1-05's D-44 pre-UAT smoke check (confirmed 2026-04-24 10:42). Without this fix, the fresh-operator UAT cannot run on a build that demonstrates the D-39 invariant across all interaction paths.

**Current (buggy) `submit()` in query_console.html (lines 81-96):**

```js
submit() {
  const [params, errors] = this.parse(this.tokenString);
  if (errors.length) { /* ... error handling ... */ return; }
  this.errorMessage = '';
  const url = this.buildUrl(params);
  htmx.ajax('GET', url, {
    target: '#view-main',
    swap: 'innerHTML',
    pushUrl: true,
  });
},
```

`buildUrl` returns a full-page URL like `/ui/tools?origin=synthesized`. The GET returns the full shell+view HTML.

**Fix: swap the entire body, not innerHTML into #view-main:**

```js
htmx.ajax('GET', url, {
  target: 'body',
  swap: 'outerHTML',
  pushUrl: true,
});
```

This matches how `hx-boost` handles nav-link clicks after Plan 10.1-02 (full-page swap via htmx's boost handler). The body swap replaces the entire page with the server-rendered response, preserving single-shell invariant.
