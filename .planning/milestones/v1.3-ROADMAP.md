## Overview

Made forge-bridge legible to its operator. Shipped a shared read-side API on `:9996` (ConsoleReadAPI sole-read-path discipline + ManifestService singleton + instance-identity gate) consumed by three surfaces — an artist-first Web UI (Jinja2 + htmx + Alpine.js, no npm), a Typer + Rich CLI companion, and MCP resources — with the synthesis manifest as a canonical artifact reachable via HTTP, MCP resource, and tool fallback shim. Phase 12 (LLM Chat) was velocity-gated mid-milestone (2026-04-23) and superseded by v1.4 FB-D, which absorbs the chat scope into a tool-call-loop endpoint.

## Milestone Summary

### Inserted Phases

- **Phase 10.1** (2026-04-23) — Artist-UX gap closure after Phase 10's D-36 dogfood UAT failed qualitatively. Six plans; all shipped 2026-04-24. Pattern lesson: hard fresh-operator UAT gate is the right tool for artist-facing surfaces; soft developer-as-operator gate is right for SSH/CI surfaces (applied at Phase 11 D-08).

### Key Decisions

- **Uvicorn asyncio task inside `_lifespan`, NOT FastMCP.custom_route** — custom_route only works in `--http` mode; stdio is the locked default and custom_route would break Claude Desktop / Claude Code. Console runs as a separate uvicorn task on `:9996` co-resident with the MCP server.
- **ConsoleReadAPI is the sole read path for ALL surfaces** — Web UI, CLI, MCP resources, future chat all consume the byte-identical `{data, meta}` envelope. Single read layer, multiple presentations. Validated three times across Phases 9/10/11.
- **ManifestService singleton injected into watcher (write path) and console router (read path)** — watcher is sole writer; console reads via `snapshot()`. Instance-identity gate (API-04) ensures `_lifespan` owns the canonical ExecutionLog and ManifestService — no duplicate instances anywhere in the process.
- **MFST-02 + MFST-03 ship in the same plan** (MCP resource + tool fallback shim together) — P-03 prevention; Cursor and Gemini CLI don't support resources, the shim costs one function.
- **Typer 0.24.1 silently drops `async def`** — verified via live test. CLI commands must be sync functions calling sync `httpx.get()`. Locked constraint.
- **Phase 11 D-08 soft UAT vs Phase 10 D-36 hard UAT** — fresh-operator UAT is the right tool for artist surfaces (Phase 10 retrospectively taught us this); developer-as-operator with "can I decipher" criterion is the right tool for technical CLI surfaces.
- **Phase 12 velocity-gated → superseded by FB-D** — explicit decision 2026-04-23, not a silent drop. CHAT-* requirements move to v1.4.
- **`Created ▼` glyph in column header** — closes Phase 10.1 HUMAN-UAT item #2 (default-sort affordance legibility) on the CLI surface immediately, even though the Web UI fix remains a deferred follow-up. Apply UX lessons to the surface that ships next.
