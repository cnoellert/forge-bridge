## The transition

Phase N+ answered "can the substrate mutate reality safely?" — the
commit primitive, the preview->apply seam. v1.7 answers the next
question: "how does a human safely ask it to?" The substrate is
mature; the operator-facing surfaces are not yet realized to the
intent they were built to serve. This milestone is the transition
from substrate construction to human-authority orchestration.

## Converged doctrine

One substrate. Two authoring lenses, plus the graph:

- **exec** — the typed-graph lens. The exact compiler: the operator
  specifies the graph directly; the compile step is deterministic
  grammar parsing.
- **chat** — the spoken-graph lens. The inferential compiler: the
  operator describes intent, the compiler infers a graph. Because
  inference can be wrong, preview is mandatory.
- **graph** — not a third authoring lens. The substrate's
  canonical self-view; exec and chat are compilers targeting it.

Both authoring lenses produce the same substrate-shape and honor the
same preview->apply seam. Chat must never silently collapse into exec
— the operator always sees what intent became what graph. That
visibility is the constitutional seam.

## Architectural law (named here, binding milestone-wide)

The law:

**Substrate self-views are first-class operator surfaces — derived, not reconstructed.**

A surface that shows the operator something about the substrate must
derive that view from substrate truth, never from a parallel artifact
that can drift. There are three such surfaces:

- graph — the substrate viewing its own execution state.
- preview manifest — the substrate viewing its own intended mutation.
- introspection — the substrate viewing its own grammar, primitives,
  and tools (Thread B opens this one).

This law binds every v1.7 phase. Thread A's preview surfaces inherit
it; Thread B's introspection surface is its first full expression. The
corollary, established at Thread B convergence: when an
operator-facing surface reveals a weakness — a vague primitive, an
ambiguous tool description — that weakness is a *substrate* defect
(incomplete self-description), not documentation debt, and is fixed in
the substrate.

## The two grammars

Operator literacy means literacy at both layers, which are distinct:

- **chain grammar** — the `->` separator, strict left-to-right
  sequencing, scoped context propagation (`_chain_parse.py`).
- **primitive grammar** — the six graph primitives (filter, if_gate,
  select, foreach, collect, commit), each a typed step-body operator
  (`graph/__init__.py`).

Thread A's NL compiler must target both layers correctly; Thread B's
introspection surface must make both legible.

## Scope — the three threads

- **Thread B — exec discoverability.** The milestone's opening phase.
  A substrate-derived introspection surface (`fbridge list/show`
  family) exposing the chain grammar, the six primitives, and the
  tools. Introspection-pure: derived from docstrings and registries,
  never a curated layer; the deliverable includes a
  docstring-quality sweep so the substrate's self-description is
  complete. Phase framing converged 2026-05-25; first code-handoff
  pending.
- **Thread A — chat input intent-layer.** The milestone's main arc.
  A genuine compile stage:
  NL -> graph-intent -> preview -> ratification -> apply.
  Enhancement, not correction (the preview seam already exists; the
  work is compelling its use). Multi-phase. Grounding (G1/G2/G3)
  complete; phase framing not yet drafted.
- Thread A and Thread B converge on one canonical source of truth:
  the substrate introspection layer Thread B builds is the
  semantic-grounding source Thread A's compiler consumes. One
  ontology, not two.

## Out of scope

- New graph primitives or grammar changes — v1.7 realizes the
  operator surfaces over the existing substrate; it does not extend
  the substrate.
- The graph lens as an authoring surface — `fbridge graph list/show`
  stays a read-only self-view.
