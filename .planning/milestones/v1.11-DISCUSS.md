## The enabling finding (grounded)

**The substrate marker the coarse router needs already exists in the tokenizer.**
`_tool_filter.py:264` defines `_NAMESPACE_PREFIXES = {flame, forge, format, synth}` —
the tool-name namespace (substrate) is already a first-class token, stripped during
matching. So "route by substrate" is **reading a marker that's already there**
(`flame_*` = Flame, `forge_*` = forge-entity), not building a new ontology. Coarse
routing is a *preference over an existing namespace axis*, not an answerability map —
which is exactly why DT's "the map doesn't exist" objection doesn't bite the coarse
path: it needs no map, only the namespace.

## Resolutions

- **Q-SR.2 — signal derivation. RESOLVED toward a deterministic, coarse signal.** The
  router needs two things, both already present: (1) the **question's substrate-class**
  — a *timeline* question is signalled by a **sequence reference** in the read ("on
  30sec_edit 21", already grounded by `_resolve_sequence_name`/24.11) plus
  timeline-attribute vocabulary; (2) the **candidate tools' substrate** — the
  `_NAMESPACE_PREFIXES` namespace. *Lean: when a read carries a sequence reference,
  prefer `flame_*` timeline candidates over `forge_*` entity candidates.* No
  per-attribute map. Plan grounds the exact vocabulary/marker set.
- **Q-SR.1 — mechanism location. RESOLVED to a concrete hook.** Not a new layer: the
  coarse preference rides in the **existing filter/narrow path** (`_step.py:255-266`)
  as a **substrate bias** applied to the candidate set *before* DI.2's arbitration —
  i.e. when the timeline signal is present, drop/deprioritize `forge_*` entity
  candidates so the `flame_*` timeline tool is what DI.2 arbitrates over. This is
  *post-compile* (operates on the compiled step's candidates) and *deterministic*, and
  it composes cleanly: **SR biases the substrate → DI.2 picks the tool → execute.**
  *Rejected:* a separate post-dispatch reroute (heavier; re-runs dispatch) and any
  compile-time model change (out of lane). Plan grounds whether the bias lives in
  `filter_tools_by_message` or `deterministic_narrow`.
- **Q-SR.3 — SR.1-T1 (timewarp test). RESOLVED: first task, prerequisite.** Daemon up;
  call `flame_get_sequence_segments` against loaded `30sec_edit 21`; confirm `file_path`
  (R8) + integer-frame duration (R10) on a real segment; **test timewarp specifically**
  (the absence claim → daemon is the authority for R9). Gates the success-criterion
  count (2/3 vs 3/3).
- **Q-SR.4 — ordinal residual. RESOLVED: minimal, inside SR.1.** "shot 10" → positional
  selection within the resolved sequence's segments; deterministic; `list[dict]`
  disambiguation when genuinely ambiguous, never a guess. Not a separate phase.
- **Q-SR.5 — R7 placement. RESOLVED: not SR.** Session/project scope is "which project
  *universe*" — session context, a different class. Its own future phase/milestone.

## The boundary holds against its own mechanism (the one thing to watch)

The substrate-bias hook sits in the *same file region* as DI.2's arbitration
(`_step.py:255-266`) — so the discipline to protect: **SR biases *which substrate's
tools* are candidates; DI.2 still picks *which tool*; neither reinterprets.** The bias
is a deterministic drop of wrong-substrate candidates, never a synthesis. If the
timeline signal is absent, SR does nothing (no bias) — it must fail safe to current
behavior, not force a substrate.

## Phase decomposition (lean — for the room/roadmap)

- **SR.1 — coarse substrate routing.** SR.1-T1 (confirm emit + timewarp, daemon) →
  substrate-bias in the filter/narrow path (timeline signal → prefer `flame_*`) →
  ordinal residual. Acceptance: **R8 + R10 answer end-to-end** against the loaded
  sequence; R9 fails *honestly* as "right substrate, no capability."
- **Carry-forwards (NOT SR phases):** R9/timewarp → capability-coverage (add a
  segment-effects read tool); R7 → session/project scope.

One phase (SR.1) + two named carry-forwards. The milestone is deliberately small —
source-of-truth correctness, no more.

## Carry-forward honesty

namespace already encodes which.* SR makes that latent axis load-bearing for routing.

not just a naming convention.
