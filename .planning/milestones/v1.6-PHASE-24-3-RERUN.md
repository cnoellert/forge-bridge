## Per-layer findings

### Layer A (runtime economics) — substantial improvement

- **Cold prefix**: 14b returned 249 tokens + 2 iters in 120s vs 32b's 0 tokens + 0 iters
- **Warm prefix 1.4× speedup** at small prompts (1.0s vs 1.4s for iter 1)
- **Warm prefix 1.6× speedup** at medium prompts (8.4s vs 13.3s for iter 2 @ 15K)
- **Warm prefix 2.0× speedup** at larger prompts (39.3s vs 80.4s for iter 3 @ 22K)
- **Prefix amplifier still present** at high prompt sizes (14b iter 5 @ 29K = 55s; gain shrinks)

The 14b swap delivers a measurable Layer A win across the prompt-size sweep. The win is larger at larger prompts (where 32b was most painful). Cold-prefix amplifier still exists but is bounded.

### Layer B (termination/convergence) — DIFFERENT failure shape; STILL broken

The 14b model exhibits a structurally different failure pattern than 32b:

- 32b: re-invokes the successful tool with identical args (loops on success without synthesizing)
- 14b: ALTERNATES between wrong affordance (`flame_find_media`) and correct affordance (`flame_execute_python`), AND re-invokes correct affordance with slight variations

Critically — across all three probes, the substrate executed `flame_execute_python` SIX TIMES with status=ok (1 cold + 2 JSON warm + 3 SSE warm). Each invocation returned the canonical Reel 1 clip list to the LLM. The model received the answer SIX TIMES. It synthesized a readable conversational response ZERO TIMES.

This is the framing's §4.3 case at its purest: **Layer B is structurally observable in isolation now that L1 and (most of) L3 hold**.

**Layer B sub-failures observed:**

1. **Affordance selection regression at iter 1.** Both 32b and 14b pick `flame_find_media` first, despite Phase 23.1's docstring rewrite of `flame_execute_python` as the canonical Flame introspection surface. The semantic-shaping work in 23.1 did not narrow iter-1 selection.

2. **Synthesis failure after successful substrate result.** Across 6 status=ok substrate dispatches, the LLM receives correct results and still does not produce a terminal conversational answer.

3. **Re-invocation of correct affordance.** Even when the model gets `flame_execute_python` right, it doesn't recognize "I have the answer; emit a terminal message." It continues iterating.

4. **LLMTOOL-04 D-07 repeat-call detection fires.** At iter 5 (JSON warm) and iter 5 (SSE warm), the model attempts `flame_find_media` for the third time with same args. The synthetic `is_error` injection at router.py:570-590 fires correctly — defense-in-depth against repetitive failure modes works as designed.

**Per framing §3.2 + §10**: Layer B structural fix is OUT OF 24.3 SCOPE. This rerun produces the cleanest precondition evidence for opening Layer B as a future phase under separate framing. The rerun does NOT close Layer B; it isolates it.

### Layer C (perceived responsiveness) — verified

- JSON probes: 120s of operator silence followed by HTTP 504 envelope
- SSE probe: first byte at 4.4ms; 13 events flow during the 120s; observable forward progress; terminal error event closes the stream cleanly

The streaming layer delivers what framing §4.0 promised — Layer C improvement orthogonal to L2 success. An operator running the SSE-aware client sees per-iter assistant + tool messages as they happen rather than 120s of silence.
