## v1.3 Requirements (final state)

### API — Shared read-side plumbing

- [x] **API-01**: ConsoleReadAPI is the sole read path for Web UI, CLI, MCP resources, and chat — Phase 9 (sole-read-path discipline validated three times across Phases 9/10/11).
- [x] **API-02**: HTTP API served on `:9996` via separate `uvicorn` task launched from `_lifespan`, alongside MCP server and watcher — Phase 9.
- [x] **API-03**: API routes under `/api/v1/` namespace returning JSON; `/api/v1/{tools,execs,manifest,health}` shipped — Phase 9. (`/api/v1/chat` deferred to v1.4 FB-D per Phase 12 supersession.)
- [x] **API-04**: `_lifespan` owns canonical `ExecutionLog` and `ManifestService` instances; instance-identity gate enforced — Phase 9.
- [x] **API-05**: Optional `StoragePersistence` read-adapter mirrors write-side Protocol shape — Phase 9.
- [x] **API-06**: Console port fallback degrades gracefully — `:9996` unavailable logs WARNING and MCP boots anyway — Phase 9.

### TOOLS — Browse registered tools with provenance

- [x] **TOOLS-01**: User can browse registered tools in Web UI with filters (origin, namespace, readOnlyHint) — Phase 10 (artist-facing column language refined in 10.1).
- [x] **TOOLS-02**: User can drill into single tool and see canonical `_meta` provenance + raw source for synth tools — Phase 10.
- [x] **TOOLS-03**: `forge-bridge console tools` subcommand lists tools with same data and filter flags as Web UI — Phase 11 (D-01 grammar mirrored 1:1).
- [x] **TOOLS-04**: `forge://tools` and `forge://tools/{name}` MCP resources + `forge_tools_read` tool fallback shim — Phase 9.

### EXECS — Execution history

- [x] **EXECS-01**: User can browse execution history in Web UI with pagination and filters — Phase 10.
- [x] **EXECS-02**: User can drill into single execution to see hash, timestamp, raw_code, intent, SQL-mirror linkage — Phase 10.
- [x] **EXECS-03**: `forge-bridge console execs` subcommand — Phase 11 (W-01 server-side `--tool` filter deferred; client-side workaround with stderr note shipped).
- [x] **EXECS-04**: Execution data reads through shared `ConsoleReadAPI` (API-01) — Phase 9.

### MFST — Synthesis manifest as canonical artifact

- [x] **MFST-01**: `ManifestService` singleton owns in-memory synthesis manifest; injected into watcher and console router — Phase 9.
- [x] **MFST-02**: `forge://manifest/synthesis` MCP resource — Phase 9.
- [x] **MFST-03**: `forge_manifest_read` MCP tool fallback shim — Phase 9 (shipped same plan as MFST-02 per locked decision).
- [x] **MFST-04**: Web UI manifest browser with sortable/filterable table + per-entry drilldown — Phase 10.
- [x] **MFST-05**: `forge-bridge console manifest` subcommand — Phase 11. **NOTE**: Phase 11 D-08 UAT flagged that manifest and tools render visually indistinguishable (column-set equivalence is by design, but operational distinction not legible). Deferred to v1.4 polish pass.
- [x] **MFST-06**: `ManifestService` satisfies EXT-01 (shared synthesis manifest between repos) — Phase 9.

### HEALTH — Bridge / router / watcher / storage status

- [x] **HEALTH-01**: Web UI health panel shows liveness for MCP, Flame HTTP bridge, WS, LLM backends, storage callback, watcher, console port — Phase 10.
- [x] **HEALTH-02**: `forge-bridge console health` subcommand — Phase 11.
- [x] **HEALTH-03**: `forge-bridge console doctor` expanded diagnostic with CI-gating exit codes — Phase 11.
- [x] **HEALTH-04**: Health surfaces in persistent compact header strip on every Web UI view — Phase 10 (nav-swap regression for the strip resolved in 10.1-01).

### CONSOLE — Web UI shell + structured query

- [x] **CONSOLE-01**: Web UI as Jinja2 + vendored htmx + vendored Alpine.js, zero JS build step — Phase 10.

- [x] **CONSOLE-05**: Every UI-touching phase includes non-developer dogfood UAT — Phase 10 + 10.1 (D-36 hard gate); pattern adapted for technical surfaces in Phase 11 (D-08 soft gate).

### CLI — Console CLI companion

- [x] **CLI-02**: CLI calls console HTTP API on `:9996` via sync `httpx`; all command bodies sync — Phase 11.
- [x] **CLI-03**: Every subcommand supports `--json` and `--help` with concrete usage examples — Phase 11.
- [x] **CLI-04**: Rich rendering when stdout is a TTY; plain text otherwise — Phase 11.
