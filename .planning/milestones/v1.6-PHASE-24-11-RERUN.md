## §1 Implementation

24.11 adds `forge_bridge/llm/resolver.py` and calls it from
`forge_bridge/console/handlers.py` after macro expansion and before tool
arbitration.

The resolver currently handles:

- `sequence_name`: `30sec 21`, `30sec21`, and `30sec_21` normalize to
  `30sec_21`
- `reel_name`: same whitespace-to-underscore normalization pattern
- `prefix` / `shot_name`: pass-through when explicitly expressed

The chat handler uses the same resolved entity map in two places:

- LLM path: enriches the last user message with a `[Resolved entities from query]`
  block.
- PR20 forced path: merges resolved entities into caller params before
  FastMCP invocation.

No tool descriptions, system prompt, model selection, registry order, or
adapter normalization changed in this phase.
